## 1. 产品哲学

### 1.1 问题

散户在投资中最缺的不是信息，而是**认识自己**。

```
你买入贵州茅台时，理由是"跌得差不多了"。
三个月后你卖出，理由是"涨不动了"。

—— 表面上这是两次独立的决策。
—— 实际上它们遵循相同的模式：你在追情绪，而不是追价值。

没有人告诉你这个模式。
Bloomberg 不告诉你，研报不告诉你，股吧也不告诉你。
```

### 1.2 解决方案

这个 Agent **不分析市场，分析用户自己**。

它不是"教练"——它不做判断、不给指令。
它也不是冷冰冰的"数据报表"——只会堆数字不会教人。

它是 **一面有记忆的镜子，也是一位从不说教的老师**：

- **镜子**：真实、不带修饰地回放用户做了什么
- **老师**：用用户自己的数据，帮用户看懂自己的行为模式

### 1.3 与"手榴弹"定位的关系

```
机构（原子弹）：10 个 CFA + Bloomberg 终端 + 自研量化系统 + 合规团队
                                ↓ 他们有人盯着你的行为
散户（手榴弹）：一句"茅台能买吗"→ 15 秒拿到结构化决策卡
                                ↓ 但没人帮你复盘自己的行为

这个 Agent 就是补这一环的：
"你自己看，这是你过去做的事，你自己判断对不对。"
```

## 2. 核心定位：既是镜子，也是最好的老师

### 2.1 镜子的职责

| 职责 | 说明 | 示例 |
|:----|:-----|:-----|
| 📝 **记录** | 忠实记录每一次用户决策的时间、标的、金额、逻辑 | "2026-06-01 买入茅台 100股 @168，理由：感觉到底了" |
| 📊 **统计** | 归纳用户的行为模式（胜率、盈亏比、持有周期、常见偏差） | "你的平均持有周期 3 天，胜率 42%" |
| 🔍 **对比** | 将当前决策与历史相似决策并列展示 | "这次和上次买五粮液很像，要不要看看当时的结局？" |

### 2.2 老师的职责

老师的输出方式不是"你应该……"，而是**用数据教用户看懂自己**：

```
镜子说：
"你的 10 笔交易中，7 笔亏损，3 笔盈利。"

老师 + 镜子说：
"你的 10 笔交易中，7 笔亏损，3 笔盈利。
 值得注意的是：3 笔盈利的平均持有 45 天，7 笔亏损的平均持有 3 天。
 你的问题不是选股，是拿不住。
 下次买入前，试试问自己一句：'如果跌 20%，我还敢加仓吗？'"

——老师给的是一句教育，不是一句指令。
```

### 2.3 不做的事

| 不做 | 理由 |
|:----|:-----|
| ❌ 不预测涨跌 | 那是大师们的活 |
| ❌ 不推荐买卖 | 用户自己决定 |
| ❌ 不评价对错 | 只展示模式和后果 |
| ❌ 不在出师前参与辩论 | 数据积累不够时，输出的对比不可靠 |

## 4. 三段式解锁设计

### 4.1 设计原则

功能渐进，但用户能感知到"它一直在观察我、越来越了解我"。不是游戏化的等级勋章，而是**像一个正在研究你的分析师**。

### 4.2 三段式

| 阶段 | 阈值 | 行为 | 前端展示 |
|:----:|:----:|:-----|:---------|
| 📝 **记录期** | 1-9 次决策 | 只记录不输出。默默建立用户行为基线 | "已记录 X 次投资行为，了解度 N%" |
| 🔍 **对比期** | 10+ 次决策 | 开始出对比报告。**这是核心价值**——用户能看到自己的模式了 | "已分析 X 次投资行为，发现 Y 个行为模式" |
| 🗣️ **出师期** | 30+ 次决策 | 可在决策卡中占一个席位（可选 toggle），以"你自己"的身份参与 | "你的投资人格已激活（可选）" |

### 4.3 解锁阈值的设计理由

| 阈值 | 理由 |
|:----|:-----|
| **10 次**才能出对比报告 | 太少样本得不出有意义的模式。5 次以内可能全是随机偏差。10 次才具备统计上的初步可信度 |
| **30 次**才能参与辩论 | 30 次 = 约 1-3 个月活跃用户的决策量。此时行为模式已经稳定，可以代表"用户的投资人格" |
| **数据不够时不输出** | 宁可不说话，不说没有根据的话。这是这个 Agent 可信度的基础 |

### 4.4 出师后的可选性

用户可以在投资分析页面自由 toggle 是否让镜子 Agent 参与：

```
☑ 巴菲特  (价值派)
☑ 林奇    (成长派)
☑ 达利欧  (宏观派)
☐ 你自己  (你的行为模式)  ← 默认关，了解度够了才亮
```

注意：**「你」也是一个大师**——不是什么伪大师，而是有着真实的交易记录和可量化行为模式的"你自己的投资人格"。在一个辩论桌上，你有权利听到"过去的自己"的声音。

## 5. 数据结构设计

### 5.1 核心数据：用户行为记录

```python
# src/memory/user_behavior.py — 行为记录存储（Phase 1+ 实现）

@dataclass
class InvestmentDecision:
    """用户的一笔投资决策记录"""
    # 基础信息
    timestamp: str                         # 决策时间
    session_id: str                        # 所属会话
    
    # 标的与方向
    ticker: str                            # 股票代码/名称
    direction: Literal["buy", "sell", "hold", "watch"]  # 操作方向
    quantity: int | None = None            # 计划数量
    price: float | None = None             # 计划价格
    
    # 用户的原始逻辑（LLM 从对话中提取）
    stated_reason: str = ""                # "感觉到底了"
    reason_category: str = ""              # "技术分析" / "消息驱动" / "情绪驱动" / "基本面分析"
    
    # 结果（后续填充）
    actual_outcome: Literal["profit", "loss", "breakeven", "pending"] = "pending"
    actual_return_pct: float | None = None  # 实际收益率
    holding_days: int | None = None         # 实际持有天数
    exited_at: str | None = None           # 卖出时间
    exited_reason: str = ""                # 卖出理由
    
    # 偏差分析（LLM 事后标注）
    biases_detected: list[str] = field(default_factory=list)  # ["confirmation_bias", "herd_mentality"]


@dataclass
class BehaviorProfile:
    """用户的行为画像（随时间累积更新）"""
    user_id: str
    total_decisions: int = 0
    
    # 整体统计
    win_rate: float = 0.0                  # 胜率
    avg_holding_days: float = 0.0          # 平均持有天数
    avg_return_per_trade: float = 0.0      # 平均单笔收益率
    max_drawdown: float = 0.0              # 最大回撤
    
    # 按方向
    buy_win_rate: float = 0.0
    sell_win_rate: float = 0.0
    
    # 按标的类型
    sector_breakdown: dict[str, int] = field(default_factory=dict)  # {"白酒": 5, "新能源": 3}
    
    # 按理由类型
    reason_breakdown: dict[str, int] = field(default_factory=dict)  # {"情绪驱动": 4, "技术分析": 6}
    
    # 常见偏差（按频率排序）
    top_biases: list[str] = field(default_factory=list)
    
    # 典型失败模式（LLM 总结）
    common_pitfalls: list[str] = field(default_factory=list)
```

### 5.2 偏差分类体系（初版）

用于 LLM 标注用户行为时使用的分类：

| 偏差类别 | 描述 | 示例 |
|:---------|:-----|:-----|
| **追涨杀跌** | 价格涨了想买，跌了想卖 | "涨了 5% 了赶紧上车" |
| **确认偏误** | 只看到支持自己观点的信息 | "研报说好，那就买"（忽略风险提示） |
| **处置效应** | 过早卖出盈利股，死扛亏损股 | "赚 3% 就卖，亏了 15% 还拿着" |
| **锚定效应** | 被历史价格锚住 | "上次到过 200，现在 180 很便宜" |
| **近因效应** | 最近涨跌影响判断 | "最近涨了，趋势肯定向上" |
| **羊群效应** | 跟风操作 | "大家都在买，我也买" |

### 5.3 与知识库的关系

用户行为数据**不存入金融知识库**，而是独立存储：

```
data/
├── knowledge_base/          ← 金融知识（所有 Agent 共享）
│   └── ...
└── user_profiles/           ← 用户行为数据（每个用户独立）
    ├── user_001/            ← 按用户 ID 分目录
    │   ├── decisions.jsonl  ← 决策流水（每行一条 InvestmentDecision）
    │   ├── profile.json     ← 行为画像（BehaviorProfile）
    │   └── reports/         ← 生成的对比报告缓存
    │       └── 2026-06-07.md
    └── user_002/
        └── ...
```

**设计理由**：
- 金融知识 = 公开数据，所有用户的 Agent 共享同一份
- 用户行为 = 隐私数据，必须隔离

## 6. 与现有架构的关系

### 6.1 复用现有组件

| 现有组件 | 镜子 Agent 如何复用 | 改造量 |
|:---------|:--------------------|:------:|
| `src/memory/knowledge_base.py` (RAG) | **不改 RAG**。用户行为数据走独立存储（JSONL），不走向量库 | 无 |
| `src/agents/base.py` (Agent 基类) | 镜子 Agent 继承 `BaseAgent`，实现 `run()` → 出对比报告 | 新增子类 |
| `AgentResult[Generic[T]]` | 输出结构化对比报告（继承 BaseModel，与 ADR-008 一致） | 新增 OutputModel |
| `src/agents/base.py` `get_tools()` | 镜子 Agent 不需要外部工具，返回 `[]` | 无 |
| `AgentContext` (辩论槽位) | 出师后通过 `peer_outputs` 参与辩论，接收其他 Agent 的输出 | 无 |
| `skill_disk.py` (Skill 插件盘) | **出师后**注册为第 8 位大师："用户自己的投资人格" | 扩展 SkillDisk |

### 6.2 新增组件

| 新增 | 说明 |
|:----|:-----|
| `src/memory/user_behavior_store.py` | 用户行为存储 & 检索（JSONL 持久化，不依赖 FAISS） |
| `src/agents/behavior_mirror.py` | 镜子 Agent 主体（记录 + 对比 + 报告生成） |
| `data/user_profiles/` | 用户行为数据目录（按用户隔离） |

### 6.3 不可触碰的原则

1. **数据隔离** — 用户行为数据不与金融知识库混存，每个用户独立目录
2. **不写死阈值** — 10 次 / 30 次 由配置控制，可调
3. **默认不激活** — 用户需要主动开启才有数据记录
4. **输出不说"你应该"** — 只展示模式，让用户自己判断

## 7. 输出形态：对比分析报告

### 7.1 对比报告的结构

```python
class BehaviorComparisonReport(BaseModel):
    """行为对比报告 —— 镜子 Agent 的输出契约（ADR-008 兼容）"""
    
    # 当前操作摘要
    current_action: str = Field(..., description="用户当前打算做的操作")
    current_reason: str = Field("", description="用户给出的理由")
    
    # 相似历史操作
    similar_actions: list[dict] = Field(
        default_factory=list,
        description="与当前操作模式相似的 N 条历史记录",
    )
    
    # 行为模式分析
    pattern_summary: str = Field(
        "",
        description="LLM 总结的行为模式，中立客观",
    )
    
    # 用户行为统计数据（仅当前维度）
    stats: dict[str, str | float] = Field(
        default_factory=dict,
        description="相关维度的统计数据，如 '该板块胜率': '33%'",
    )
    
    # 一句"老师"的话
    teaching_note: str = Field(
        "",
        description="基于行为模式的教育性提示，不是指令，而是引发思考",
    )
    
    # 置信度（数据充分程度）
    confidence: float = Field(
        0.0,
        ge=0.0,
        le=1.0,
        description="基于相似案例数量的置信度",
    )
```

### 7.2 对比报告的触发时机

| 场景 | 触发方式 | 说明 |
|:----|:---------|:-----|
| 用户主动查看 | 点击"我的行为分析" | 完整报告 |
| 用户搜索标的时配带 | 可选 toggle | 简短版，只对比相关历史 |
| 用户做出决策后 | 自动记录 + 可选弹窗 | "你这次决策和你之前的 X 次类似" |

> **新增事项详解**（基于 §10 技术风险研究）：
> - **偏差检测规则引擎**：覆盖持有 < 7 天 → 短期交易、买入后 3 天内卖出 → 处置效应嫌疑、同标 T+0 → 日内交易、连续 3 次同向操作 → 趋势追逐。这些不需要 LLM，100% 确定性标注
> - **独立校验器**：使用不同模型或不同 prompt 复核 LLM 标注器的输出。一致才标注为"确定"，否则降级为"疑似"或"提示"
> - **冷启动阶段 A**：匹配不到相似案例时不返回空，改为展示板块级统计概览 + 匿名化用户组对照数据
> - **Wrapped 定期报告**：在数据积累到 10+ 条后自动生成。形式为月度/周度行为摘要，包含胜率、持有周期、板块分布和关键趋势变化

### Phase 2+：深度行为分析

| 事项 | 说明 | 备注 |
|:----|:-----|:-----|
| 行为模式演进追踪 | "你的追涨倾向比上个月改善了" | |

| **决策干预层** | 用户做出重大偏离历史模式的操作时，弹出轻量提示 | 基于 Generative Pause 三原则，默认关闭 |
| 与其他用户比较 | 匿名化："你的持仓周期比 80% 的用户短" | |
| **偏差检测 Embedding 升级** | 从规则 + LLM 升级为 embedding 语义相似度 + LLM 的混合方案 | 基于 FinBias 数据集和 FBI 框架 |

## 9. FAQ

**Q：用户行为数据的安全性怎么保证？**
A：数据存在本地 `data/user_profiles/` 目录，按用户 ID 独立目录隔离。不上传任何用户行为数据到外部服务。用户可随时清空自己的行为数据。

**Q：10 次和 30 次是硬编码吗？**
A：不是。这两个阈值由配置控制，可根据产品运营需要调整。10 次记为 `MIN_COMPARISON_THRESHOLD`，30 次记为 `MIN_DEBATE_THRESHOLD`。

**Q：怎么判断两次操作"相似"？**
A：初版用规则匹配（同板块、同方向、类似理由关键词）。后续可升级为 embedding 语义相似度。

**Q：如果用户只有买入记录没有卖出记录怎么办？**
A：`actual_outcome` 默认为 `"pending"`，对比报告会标注"该笔交易尚未了结"。不影响模式分析。

**Q：偏差检测准确吗？**
A：初版基于 LLM 标注 + 规则兜底（如持有 < 7 天自动标"短期交易"）。LLM 标注的偏差作为"提示"而非"诊断"展示给用户，措辞用"我发现……"而非"你是……"。

**Q：这个 Agent 会占用辩论引擎的 Slot 吗？**
A：出师后才会占据一个可选 Slot，且默认关闭。用户需要主动勾选"你自己"才会参与。不勾选不影响原有辩论流程。

## 10. 技术风险与缓解方案

> 本章节基于 2025-2026 年学术研究成果与行业实践，对设计中的三个核心风险进行技术调查，并给出经过验证的缓解方案。

### 10.1 风险一：LLM 偏差检测的可靠性

#### 10.1.1 问题

同一个 LLM 既做用户行为分析、又做偏差标注，存在**自我确认偏误（self-confirmation bias）** 和**叙事偏误（narrative bias）** 的风险——模型倾向于生成连贯但未必准确的行为解释，而不是真实反映用户模式。

#### 10.1.2 研究结论

2025-2026 年多项研究确认了这一风险，并提出了成熟的对策：

| 研究 | 来源 | 核心发现 |
|:-----|:-----|:---------|
| **FBI 框架**（Financial Bias Indicators） | ACL 2025 Findings | 四组件框架（Bias Unveiler / Detective / Tracker / Antidote），使用金融因果知识 **68% 偏差消减** |
| **FinBias 数据集** | ACM 2025 | 11 类偏见评测 + SAFE 评分，验证 prompt engineering 缓解策略有效 |
| **"金融大脑扫描"**（Sparse Auto-Encoder） | arXiv 2508.21285, 2025 | 定位 LLM 金融推理中的具体偏差特征，负向引导可实现 Sharpe 4.28（基线 3.87） |
| **行为经济学 LLM 偏差** | NBER Working Paper 34745, 2026 | 系统性实验证明：理性提示可有效降低 LLM 在信念型任务中的偏差 |
| **金融 LLM 评估五宗罪** | arXiv 2602.14233 | 识别前瞻偏差、幸存者偏差、叙事偏差等 5 类系统性偏差 |

**关键结论**：金融 LLM 的偏差是可检测、可缓解的，但需要系统性的工程方案，不能依赖单一模型的"自觉"。

#### 10.1.3 缓解方案：双引擎校验架构

采用**规则引擎 + LLM 标注器 + 独立校验器**三层架构，而非单一模型全权负责：

```
输入: 用户决策记录
    │
    ▼
┌─────────────────────────────────┐
│  规则引擎（第一层·确定性）         │
│  · 持有 < 7 天 → 标记"短期交易"     │
│  · 买入后 3 天内卖出 → "处置效应嫌疑" │
│  · 同标的 T+0 买卖 → "日内交易"     │
│  · 连续 3 次同向操作 → "趋势追逐"    │
│  输出: 确定性标注（可信度 100%）     │
└─────────────────────────────────┘
    │ 规则不覆盖的部分
    ▼
┌─────────────────────────────────┐
│  LLM 标注器（第二层·推断）         │
│  · 分析用户 stated_reason         │
│  · 使用理性提示降低自身偏差         │
│  · 输出偏差分类 + 置信度           │
│  输出: 推断标注（带置信度分数）      │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  独立校验器（第三层·对抗）         │
│  · 使用不同 prompt / 不同模型      │
│  · 对 LLM 标注器的输出进行复核     │
│  · 标注一致性 < 阈值 → 降级为      │
│    "疑似"而非"确定"               │
│  输出: 最终标注 + 可信度标签       │
└─────────────────────────────────┘
```

**具体实现要点**：

- **规则引擎**：覆盖持有周期、同标的重复操作、日内回转等可由时间和价格数据直接判断的场景。规则输出不需要 LLM，可信度 100%
- **LLM 标注器**：使用 DeepSeek（项目主力模型），prompt 中嵌入 FBI 框架的 Bias Antidote 策略——要求模型先列举反方证据再做判断
- **独立校验器**：使用更小/更便宜的模型（如 DeepSeek-Tiny 或量化版本），使用完全不同的 prompt 模板。两者一致才标注为"确定"，不一致标注为"疑似"并在报告中说明

#### 10.1.4 用户展示原则

```
规则确定       → 直接展示（"你持有的最短记录是 2 天"）
LLM+校验一致   → 标注为"发现"（"我发现你有…的倾向"）
LLM+校验不一致 → 标注为"提示"（"这可能是…你感觉呢？"）
规则+LLM 都不  → 不输出（宁可不说话，不说没根据的话）
```

### 10.2 风险二：相似行为匹配的冷启动

#### 10.2.1 问题

用户数据积累不足时（1-9 次决策），规则匹配要么空结果（用户失望），要么匹配太宽泛（用户觉得没用）。这是行为分析系统最常见的冷启动问题。

#### 10.2.2 研究结论

| 方法 | 来源 | 适用阶段 |
|:-----|:-----|:---------|
| **Session 相似性框架**（上下文+时间特征） | Decision Support Systems 2024 | 零历史数据时，基于当次会话特征匹配 |
| **预训练金融 Embedding**（`Finance2_embedding_small_en-V1.5`） | HuggingFace, 余弦准确率 97.9% | 10-50 条记录时，384 维嵌入即可工作 |
| **元学习（MAML/PAML）** | arXiv 2012.11842 | 50+ 用户后可进行跨用户迁移学习 |
| **多特征融合哈希 + HNSW 索引** | IEEE BigComp 2025 | 适合生产环境的高效相似检索 |

**关键结论**：冷启动问题在学术和工程上都有成熟方案。核心思路是**分阶段升级匹配策略**，而不是试图一个方案覆盖所有阶段。

#### 10.2.3 缓解方案：三阶段渐进匹配

```
Phase 用户数据量    匹配策略                       输出风格
───── ──────────── ──────────────────────────── ──────────────────
A     0-9 次      规则匹配（同板块/同方向）        标注"暂无足够相似案例"
                  + 当次上下文特征                展示板块级统计概览
                  + 用户组对照（匿名化聚合）        "该板块用户平均持有…"

B     10-29 次    Embedding 语义相似度            展示 2-3 条最相似历史
                  (sentence-transformers)        标注相似度分数
                  384-dim → HNSW 索引             置信度 < 0.7 时加"弱相似"标记

C     30+ 次      Embedding + LLM 相似性排序      展示完整对比报告
                  偏差检测启用                    行为模式总结上线
                  三段式全功能                     出师 toggle 可用
```

**阶段 A 的冷启动策略详解**：

1. **规则匹配降级但不消失**：匹配不到高相似度历史时，不返回空结果，而是展示板块级统计

```
示例输出（冷启动期）：
"你正在查看贵州茅台（白酒板块）。
 当前你可比历史：暂无足够相似操作。
 你的白酒板块记录：2 笔（1 赚 1 亏），平均持有 5 天。
 该板块用户中位数持有：45 天。—— 仅供参考。"
```

2. **基于当次会话的上下文特征匹配**：不依赖用户历史，而是提取当前操作的 N 个特征维度（标的板块、操作方向、理由关键词情感），在匿名聚合用户群中寻找类似模式

3. **通用金融 Embedding 即可工作**：`Finance2_embedding_small_en-V1.5`（384 维，97.9% 余弦准确率）针对小样本金融场景优化，BGE-small 级别的模型即可嵌入用户 short text

#### 10.2.4 预训练模型选择

| 模型 | 维度 | 数据集规模 | 余弦准确率 | 推荐场景 |
|:-----|:----|:----------|:----------|:---------|
| `baconnier/Finance2_embedding_small_en-V1.5` | 384 | 15,525 | 97.9% | Phase B 主力，轻量级 |
| `BAAI/bge-small-en-v1.5` | 384 | 通用 | — | 英文本地化替代 |
| `shibing624/text2vec-base-chinese` | 768 | 中文通用 | — | **中文场景首选** |

**技术要点**：
- 384 维嵌入即可冷启动，不需要大模型
- 相似度检索使用 HNSW 索引，时间复杂度 O(log n)，扩展到 10 万条记录也只需毫秒级
- 用 `MultipleNegativesRankingLoss` 在自己的小样本上微调，100 条 triplet 即可有明显改善

### 10.3 风险三：用户是否会真的反思？

#### 10.3.1 问题

产品核心假设是"用户愿意面对自己的行为模式并反思"。行为金融学研究支持这个方向的合理性，但产品层面存在实际障碍：散户的典型心理是**赚钱归自己、亏钱怪市场**，主动复盘需要较高的认知意愿。

#### 10.3.2 研究结论

| 研究/实践 | 来源 | 核心方法 | 镜子 Agent 可借鉴点 |
|:----------|:-----|:---------|:-------------------|
| **Financial Checkpoints** | FCAC Canada 2025 冠军论文 | 在交易流程中嵌入"检查点"——交易前弹出上下文消费反馈 | 用户决策前弹出行为回顾 |
| **Impulse Check** | Devpost 2025 | AI 分析产品 URL → 显示机会成本（"这笔钱 10 年后值多少"） | 用用户自己的历史数据计算机会成本 |
| **Designed Skepticism** | Yonsei/KCI 2025 | nudge + skepticism 的平衡设计 | 对比报告同时展示正反两面数据 |
| **Generative Pause** | Versions.com 2025 | 三原则：Conscious Verification / Symmetric Benefit / Proportional Calibration | 在决策点而非复盘点干预 |
| **Wrapped 范式** | Financial Express 2025 | 200M 用户 24 小时内参与，19% YoY 增长 | 定期（周/月）行为总结 |
| **Nudge 在金融 App** | Zenodo 2025 研究 | 映射常见偏差到具体数字 nudge 干预 | 行为模式 → 具体提示的对应表 |

**关键结论**：
1. 用户不会主动来找镜子——产品需要**把镜子搬到用户面前**
2. "速度 bump"模式（在决策点插入摩擦）已被验证比事后报告更有效
3. Wrapped 范式证明了**定期可视化总结**的巨大情感共鸣和传播力
4. 透明度和可控性是用户接受这些干预的前提

#### 10.3.3 缓解方案：三层触发机制

```
被动记录层（自动，无感知）
┌─────────────────────────────────────┐
│ · 每次用户搜索/浏览/买入/卖出 → 自动记录  │
│ · 用户不需要知道它在工作               │
│ · 隐私声明在首次使用时一次性说明          │
└─────────────────────────────────────┘

定期回顾层（推送式，低频率）
┌─────────────────────────────────────┐
│ · 周报/月报："你的投资行为摘要"          │
│ · Wrapped 风格可视化（胜率/持有周期/     │
│   最多交易的板块/最大亏损原因）          │
│ · 只在用户活跃时推送（减少打扰）          │
│ · 默认开启但有开关                     │
│ · 样式参考：Lloyds 银行年度消费报告      │
└─────────────────────────────────────┘

决策干预层（可选，中频率）
┌─────────────────────────────────────┐
│ · 用户做出重大决策时（切换标的类型/        │
│   大幅偏离历史模式）弹出轻量提示          │
│ · 不是"对比报告"而是"你的模式提醒"        │
│ · 样式："你这次准备买入科技股。             │
│   过去 3 次买入科技股，2 次亏损。          │
│   和之前不太一样，提醒一下。 —— 镜子"      │
│ · 默认关闭 → 用户感受到价值后主动开启      │
│ · 基于 Generative Pause 三原则设计       │
└─────────────────────────────────────┘
```

**具体的 Wrapped 报告结构设想**：

```
┌─────────────────────────────────────┐
│  📊 你的月度投资行为报告（6月）           │
│                                     │
│  你本月交易了 12 次                    │
│  涉及 5 个板块                       │
│                                     │
│  你的胜率 42% → 比上月提升 5% 🟢      │
│  平均持有 4 天 → 比上月缩短 1 天 🔴    │
│                                     │
│  本月亮点：                           │
│  你第一次使用了"止损"策略 ✅            │
│                                     │
│  本月最值得注意的模式：                  │
│  "你有在下跌 3% 后加仓的倾向，            │
│   这种情况 70% 后续会继续下跌。          │
│   下次跌了先等等？"                     │
│                                     │
│  [查看完整对比报告]                     │
└─────────────────────────────────────┘
```

#### 10.3.4 采用原则

1. **不要在用户亏钱时推送**——通过检测用户会话情绪，亏损状态只记录不推送报告
2. **先展示进步再展示问题**——每份报告以正面或中性信息开头
3. **用户始终可关闭**——任何推送性内容都有明确的关闭入口
4. **渐进启用**——默认只做被动记录，用户在首次看到对比报告价值后，自主解锁更多功能

### 10.4 风险总结

| 风险 | 风险等级 | 缓解方案 | 验证状态 |
|:-----|:--------|:---------|:---------|
| LLM 自我确认偏误 | 🔴 高 | 双引擎校验架构（规则 + LLM + 独立校验） | ✅ ACL 2025 FBI 框架验证 68% 偏差消减 |

| 用户主动反思不足 | 🟡 中 | 三层触发机制（被动记录 + 定期回顾 + 决策干预） | ✅ FCAC 2025 / Wrapped 范式验证 |
