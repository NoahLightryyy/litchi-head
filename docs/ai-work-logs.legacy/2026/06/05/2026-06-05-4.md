根据 `docs/技术实现方案-增强补充.md` 补充 G 的 Mock 方案设计，实现了：

**`tests/conftest.py`**（三个工厂函数 + 六个 fixtures）：
| 组件 | 说明 |
|------|------|
| `make_mock_llm_service(text_response)` | 创建固定响应的 mock LLMService |
| `make_mock_llm_sequence(responses)` | 创建按顺序返回不同响应的 mock |
| `make_mock_llm_error(exception)` | 创建调用时抛异常的 mock（测试错误路径） |
| `mock_llm` fixture | 自动 patch `src.utils.llm.llm_service` |
| `mock_llm_sequence` fixture | 序列响应 mock |
| `mock_llm_error` fixture | 异常 mock |
| `sample_context` fixture | 标准 AgentContext |
| `sample_result` fixture | 标准 AgentResult |
| `sample_message` fixture | 标准 AgentMessage |

**技术细节**：
- mock 精确匹配 `LLMService` 的 `ainvoke()` 和 `invoke_structured()` 方法签名
- `invoke_structured` 按传入的 `output_model` 类型自动创建对应 Pydantic 实例
- 使用 `unittest.mock.AsyncMock` + `patch`，与 pytest 生态无缝集成
- 支持自定义异常注入（`ConnectionError`, `ValueError` 等）
- `pytest_collection_modifyitems` 自动为协程测试函数添加 `asyncio` marker
- 注册了 `unit`, `integration`, `slow` 三个自定义 marker
- 修复了 `asyncio.iscoroutinefunction` → `inspect.iscoroutinefunction`（Python 3.14 弃用警告）
