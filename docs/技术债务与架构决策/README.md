# 📋 技术债务与架构决策

> 两个并行的工程管理流程：
> - **技术债务日志** — 记录已知的质量问题、分类分级、跟踪修复
> - **架构决策记录** — 记录关键的选型决策、理由和舍弃方案

## 工作流程

```mermaid
graph LR
    A[发现债务/做决策] --> B{是什么?}
    B -->|质量问题| C[登记到技术债务日志]
    B -->|技术选型| D[记录到架构决策记录]
    C --> E[评估优先级]
    E --> F[排期修复]

```
