### 战线上谁在做什么

```
                    ┌── 中国实盘路线 ── vnpy (37k⭐)
                    │    事件驱动，40+交易所接口（CTP/XTP/恒生）
                    │    中国量化实盘的事实标准
                    │
  全流程量化交易 ──┼── 全球云路路线 ── QuantConnect/LEAN (20k⭐)
                    │    云端原生，130+市场，C#核心
                    │    数据质量最高（400TB无幸存者偏差）
                    │
                    └── 加密市场路线 ── Freqtrade (44k⭐) / CCXT
                          批量交易所对接（通过 CCXT 统一接口）
                          加密货币赛道最成熟
```

## 竞品分析

| 项目 | ⭐ | 战线角色 | 值得看的 |
|------|:---:|---------|---------|
| **vnpy** | 37k | 中国实盘标准 | CTP / XTP / 恒生等 40+ 接口封装、事件驱动引擎 |
| **QuantConnect** | 20k | 全球云路线 | Broker 适配层抽象设计（多券商统一 API） |
| **Freqtrade** | 44k | 加密路线 | CCXT 统一交易所接口、订单状态管理 |

### 关键启示

| 来自谁 | 启示 |
|--------|------|
| **vnpy** | 你的最终实盘对接目标。事件驱动引擎+40+接口封装，中国最全 |
| **QuantConnect** | Broker 适配层设计——多券商统一抽象的架构思路 |
| **Freqtrade** | 加密方向的选择。CCXT 统一接口设计值得借鉴 |

## 研究问题

- [ ] 实盘接口 vs 模拟接口的抽象：如何让辩论模块不关心底层是模拟还是实盘？
- [ ] 订单状态管理（提交→确认→成交→失败→撤销）的状态机设计？
- [ ] 滑点模型：回测时的滑点假设 vs 实盘的真实滑点？
- [ ] 交易成本：A 股佣金/印花税/过户费怎么算进决策？
- [ ] 接口失败处理：交易所连接断开、订单超时、撤单失败怎么办？
- [ ] 交易日志：每笔交易的全链路追踪（谁决策的、谁批准的、谁执行的、结果如何）？
