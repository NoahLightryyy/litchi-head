# 功能模块：Agent 人格与提示词工程

> 大师 Agent 的身份编码层 — 定义"每个角色是谁、怎么思考、如何决策"

## 模块定义

大师 Agent 的角色设定、投资哲学编码、提示词策略、系统提示词管理。

**职责边界**：
- ✅ 各大师角色的人格定义（投资风格、决策原则、风险偏好）
- ✅ 系统提示词的编写与版本管理
- ✅ 提示词策略（角色设定、输出格式、思维链引导）
- ✅ 不同 LLM 的提示词适配（DeepSeek vs Claude vs GPT）
- ❌ 不负责 Agent 之间的编排（那是 Agent 编排模块的事）
- ❌ 不负责 Agent 的记忆管理（那是记忆模块的事）

## 代码结构

| 文件 | 说明 |
|------|------|
| `src/agents/master_agent.py` | MasterAgent 通用实现，通过配置参数区分角色行为 |
| `src/agents/base.py` | AgentContext，提示词构建基础 |

## 架构（当前状态）

MasterAgent 采用通用角色 + 配置化行为的设计，比硬编码多个 Agent 类更灵活。角色行为通过配置参数驱动，同一套代码支撑所有大师。

当前阶段需加强的是**人格的鲜明度**——每个大师的决策规则是否足够具体、可执行？目前人设偏笼统（如"你是价值投资者"），缺少 AI Hedge Fund 那种可执行决策规则（如"安全边际至少 30%"）。

## 数据契约（关键模型）

（暂无独立 Pydantic 模型；角色配置通过 MasterAgent 配置参数驱动，未抽象为独立数据层）

## 下一步

1. **人格规则具体化** — 梳理所有大师提示词，统一模板格式，将笼统描述替换为具体、可执行的决策规则
2. **提示词版本追踪** — 建立提示词版本管理机制，每次变更可追溯影响

4. **信息隔离评估** — 研究 TradingAgents 的分层提示词策略（分析师/研究员/交易员/风控各层信息隔离），判断是否适用于 litchi-head 场景
