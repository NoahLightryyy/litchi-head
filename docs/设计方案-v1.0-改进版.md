## 第一章 · 产品愿景与定位

### 1.1 产品愿景

打造一个融合**多智能体协同**、**用户共创**、**开源生态**与**自动化交易**的下一代投资辅助平台。通过多个专业 AI Agent 协作采集新闻、行情、分析数据，连接同花顺实盘/模拟账户，在用户可控的前提下提供深度投研、智能托管与社区化策略分享。让专业级量化分析与主观交易经验，以极低的使用门槛触达每一位投资者。

### 1.2 产品定位

> **中国首个开源的多智能体投资协作平台 —— 让每个投资者拥有自己的 AI 投研团队**

### 1.3 核心差异化主张

| 对标竞品 | 我们的差异化价值 |
|---------|---------------|
| vs TradingAgents（纯美股/技术驱动） | **中国市场适配 + 用户共创生态 + 教育普及** |
| vs AI Hedge Fund（大师人格/全球数据） | **用户可微调自己的AI人格 + 人机协作 + 开源** |
| vs Vibe-Trading（零代码/多回测） | **四组辩论深度 + 教育Agent + 社区市场 + A股生态** |
| vs GIM/TIFIN（商业化闭源） | **开源社区驱动 + 个人投资者普惠** |

### 1.4 目标用户群体

- **个人股票/基金投资者**（核心用户）— 使用智能推荐、托管交易、教育助手
- **投资爱好者/交易员** — 参与辩论、构建Skill、训练AI人格
- **量化开发者** — 基于开源引擎二次开发、贡献社区
- **金融机构** — 企业版私有化部署

### 2.3 多Agent工作流核心逻辑（v1.0 升级版）

1. **事件触发**（定时/用户请求/行情异动）
2. **调度器并行拉起**：新闻分析Agent、行情分析Agent、同花顺账户Agent（确认连接）
3. **风控预检** ★新增★：风控Agent检查当前风险敞口，确认允许新交易
4. **大师顾问团并行产出** ★新增★：8-10位大师Agent独立输出意见
5. **数据就绪后，启动四组辩论Agent群**：每组3分析师+1评审，增加强制质疑和跨组交叉质疑
6. **辩论结果 + 大师意见汇总**：交由融合排序服务，结合用户风险偏好与资金生成最终推荐
7. **风控终审** ★新增★：风控Agent对最终方案进行一票否决校验
8. **若开启托管**：交易决策发送至同花顺执行Agent，执行结果返回
9. **反思触发** ★新增★：交易结束后触发反思机制，存入长期记忆

## 第三章 · Agent 角色定义

### 3.1 基础Agent角色总表（v1.0 升级版）

| Agent | 职责 | 技术实现 | 变更状态 |
|-------|------|----------|---------|
| **新闻分析Agent** | **从"采集"升级为"深度分析"**：多时间窗口加权分析(3d/10d/30d)，新闻分类(宏观/行业/公司事件)，实体关系抽取(人物/公司/产品关联)，谣言鉴别（交叉验证信源） | Scrapy/Playwright + NLP情感模型 + LLM深度摘要 + 实体关系抽取 | ⬆️ 升级 |
| **行情分析Agent** | 连接股市数据源（新浪/聚宽/交易所Level2），获取实时报价、K线、技术指标、板块资金流。**新增**：输出技术面分析结论（MACD/RSI/布林带形态识别） | Websocket行情网关 + 时序数据库 + 技术指标计算引擎 | ⬆️ 增强 |
| **同花顺账户Agent** | 安全桥接用户同花顺账户，查询持仓/可用资金，执行买卖指令，返回成交信息 | 同花顺OpenAPI / 模拟交易协议，金融安全沙箱 | ➡️ 不变 |
| **教育助手Agent "小智"** | 基于大模型+RAG，解释MACD/KDJ/PE等指标，梳理行业上下游与热点板块。**新增**：与大师Agent联动，用户可"请教"大师解释概念 | LangChain + 向量库（指标/行业研报/大师语料） | ➡️ 不变（独特亮点）|
| **交易策略Agent** | 依据用户配置的skill/人格，或系统辩论结果，生成最终交易指令。**新增**：接受风控Agent的否决校验 | 策略运行时，连接回测引擎和执行层 | ⬆️ 增强 |
| **风险控制Agent** | **从"风控层"升级为独立Agent**：实时监控组合风险，计算VaR/波动率/最大回撤预期，设置动态止盈止损。**独立于交易决策流程**，拥有**一票否决权**，生成每日独立风控报告 | 波动率模型 + VaR计算 + 组合优化器 | 🆕 新增 |
| **投资大师人格Agent池** | 8-10位知名投资大师的人格Agent，每位大师独立分析标的，输出独立看法+评分。大师意见作为额外信号输入辩论评审环节 | LLM提示词工程 + 语料向量库RAG（LangChain） | 🆕 新增 |

### 3.2 投资大师人格Agent池（新增 · P0级）

#### 3.2.1 设计思路

将8-10位世界知名投资大师的投资哲学编码为独立Agent，每位大师Agent：

- **独立分析**当前标的，输出买入/持有/卖出的独立判断
- 输出**置信度评分**（1-10）+ **推理逻辑链** + **引用语料来源**
- 大师意见作为**额外信号**输入四组辩论的评审环节
- 用户可"**请教**"某位大师，查看其历史推荐记录和"战绩"

#### 3.2.2 大师名单（建议）

| 大师 | 核心投资哲学 | 语料来源 | 覆盖风格 |
|------|-------------|---------|---------|
| 沃伦·巴菲特 | 价值投资、安全边际、护城河 | 致股东信(1977-2024) | 价值型 |

| 本杰明·格雷厄姆 | 净现值、低估策略、安全边际 | 《聪明的投资者》《证券分析》 | 深度价值 |
| 彼得·林奇 | 成长价值、PEG、日常观察 | 《战胜华尔街》《彼得·林奇的成功投资》 | 成长型 |
| 乔治·索罗斯 | 反身性理论、趋势博弈 | 《金融炼金术》、索罗斯演讲 | 宏观对冲 |

#### 3.2.3 大师Agent技术实现

- **提示词工程**：系统提示词编码大师的核心投资原则、经典语录、方法论框架
- **语料RAG**：将大师公开著作和演讲向量化存入 Milvus，检索增强生成
- **输出格式**：`{ "action": "buy/hold/sell", "confidence": 1-10, "reasoning": "...", "citation": "..." }`
- **"战绩"追踪**：记录每位大师的历史推荐准确率，展示"近3个月/6个月/1年胜率"

#### 3.2.4 大师Agent的调用时机

| 场景 | 触发条件 | 作用 |
|------|---------|------|
| **日常推荐** | 每次辩论启动 | 与四组辩论**并行运行**，结果作为补充参考 |
| **用户主动请教** | 用户点击任一大师头像 | 进入深度对话模式，用户可与大师讨论具体标的 |
| **争议裁决** | 四组辩论评分接近（最高-最低<2）| 自动启动大师投票，加权计入最终排序 |
| **教育联动** | 用户在教程中触发概念 | 大师用其特有的语言风格解释概念 |

## 第四章 · 核心辩论工作流（强化对抗版）

### 4.1 四组辩论架构总览

保留原有的四组并行辩论架构：

```
输入: 当前持仓、资金、偏好(激进/保守)、新闻聚合、行情快照、大师意见
                              ↓
并行启动四组 (代表四种投资哲学):
  [价值组]   [成长组]   [趋势组]   [量化对冲组]
```

### 4.2 辩论流程详细设计（v1.0 增强版）

新流程在原方案基础上增加了**强制质疑轮**和**跨组交叉质疑轮**：

```
第1步：分析师独立提案（保留原方案）
   → 组内3位分析师分别从不同维度产出初始方案
   → 每位分析师输出：标的 + 仓位 + 核心理由 + 预期收益/风险

第2步：强制质疑轮 ⭐新增⭐
   → 分析师A必须找出分析师B方案中的至少3个漏洞
   → 分析师B逐一回应（承认并修正 / 反驳并给出证据）
   → 分析师C做"裁判式"对比：评估A和B方案的优劣
   → 此轮禁止"同意"或"+1"，必须提出建设性批评

第3步：跨组交叉质疑 ⭐新增⭐
   → 价值组可以挑战斗争议点（如质疑趋势组的追涨风险）
   → 被挑战组在2轮对话内回应
   → 系统自动识别跨组争议焦点，生成分歧热力图
   → 跨组质疑记录作为评审模型的额外输入

第4步：评审模型总结（保留原方案，扩充）
   → 评审模型综合：组内辩论 + 跨组质疑 + 大师意见 ⭐新增⭐
   → 输出融合方案 + 评分(1-10) + 3个关键词 + 争议点摘要 ⭐新增⭐
```

### 4.3 大师Agent的辩论介入点

- **争议裁决**：当组内分析师分歧过大（评分标准差>3），自动邀请相关风格的大师参与
- **跨组争议仲裁**：跨组交叉质疑中无法达成共识的焦点，由相关大师投出参考票
- **双向反馈**：辩论结论同步给大师Agent，大师可据此调整后续分析

### 4.4 辩论结果输出格式

| 输出项 | 说明 | 新增 |
|-------|------|------|
| 每组最终方案 | 标的 + 仓位 + 理由 | — |
| 综合评分(1-10) | 评审模型给出的评分 | — |
| 三个关键词 | 如"高安全边际、政策驱动、超跌反弹"| — |
| **争议点摘要** | 组内/跨组的主要分歧焦点 | 🆕 |
| **置信度** | 评审模型对方案的确信程度(0-1) | 🆕 |
| **大师一致性指数** | 大师团对本组的平均支持率 | 🆕 |
| **分歧热力图** | 哪些标的/板块Agent之间争议最大 | 🆕 |

### 4.5 示例提示词框架（更新）

**分析师提示词（新增强制质疑指令）**：
```
你是一名{组名}分析师。当前辩论记录：{debate_log}
请对分析师{对手}的方案提出至少3个质疑点。
要求：具体、有数据支撑、直指风险漏洞。
输出格式：
- 质疑点1：[质疑内容] + [理由/数据]
- 质疑点2：[质疑内容] + [理由/数据]
- 质疑点3：[质疑内容] + [理由/数据]
```

**评审模型提示词（扩充）**：
```
你是一个资深投资评审员，负责评估三位分析师的辩论。
辩论记录：{debate_log}
跨组质疑记录：{cross_group_challenges}
大师意见参考：{master_opinions}

请综合他们的观点，输出：
1. 最终投资组合方案（股票代码、仓位、理由）
2. 综合评分（1-10）
3. 三个关键词（如"高安全边际""政策驱动""超跌反弹"）
4. 争议点摘要（主要分歧在哪里，如何处理的）
5. 置信度（0-1）
```

### 4.6 辩论质量评估机制（新增）

- **事后评估**：将每组辩论胜出方案与实际市场结果对比，计算命中率
- **分析师Elo评分**：跟踪每位分析师的"命中率"和"辩论贡献度"，优胜者获得更高权重
- **辩论质量分**：基于分歧度、逻辑完整性、引用数据量等维度评估辩论质量
- **群体智能进化**：每一次辩论 × 市场结果 = 训练数据，持续优化模型

## 第五章 · 记忆系统设计（新增章节）

### 5.1 记忆系统总体架构

```
┌──────────────┐  ← Token窗口内，session级别，随session结束清空
│  工作记忆     │
│ (Working Mem)│  → Redis缓存 + LLM上下文窗口
├──────────────┤  ← 滑动窗口30天，近期交易得失总结
│  情景记忆     │
│ (Episodic Mem)│ → PostgreSQL（结构化）+ Milvus（向量化摘要）
├──────────────┤  ← 跨session持久化，用户画像+Agent经验
│  长期记忆     │
│ (Long-term    │  → Milvus向量库 + PostgreSQL元数据
│  Memory)      │
├──────────────┤  ← 每笔交易结束后触发，存入长期记忆
│  反思机制     │
│ (Reflection)  │  → LLM生成反思摘要 → 向量化存入Milvus
└──────────────┘
```

### 5.2 工作记忆（Working Memory）

- **内容**：当前辩论session的全部上下文
  - 本次请求输入（持仓/资金/用户偏好）
  - 各Agent的中间输出
  - 辩论对话历史（含强制质疑和跨组质疑记录）
- **存储**：LLM上下文窗口内 + Redis缓存（前端轮询）
- **生命周期**：session级别，结束后归档至MongoDB

### 5.3 情景记忆（Episodic Memory）

- **内容**：近期交易的得失总结
  - 最近30笔交易（或滑动窗口30天）
  - 每笔交易：时间/标的/方向/盈亏/事后反思
  - 用户近期反馈记录（👍/👎、手动修改订单）
- **存储**：PostgreSQL（结构化）+ 向量化摘要（Milvus）
- **检索**：每次辩论启动时自动加载，供分析师Agent参考
- **更新**：每笔交易结束后追加，超出窗口则归档

### 5.4 长期记忆（Long-term Memory）

- **内容**：跨session沉淀的交易经验和用户画像
  - 用户长期偏好向量（风险偏好、行业偏好、调仓习惯）
  - Agent积累的交易经验（"这个股票在XX条件下表现好"）
  - 大师Agent的语料向量库
  - 历史辩论中的高评分策略模式
- **存储**：Milvus向量库 + PostgreSQL元数据
- **检索**：语义检索 + 时间衰减权重

### 5.5 反思机制（Reflection）

- **触发时机**：每笔交易结束后（或每日收盘后批量触发）
- **反思过程**：
  - **输入**：交易记录、决策过程摘要、实际盈亏、市场状态
  - **反思**：决策正确/错误的原因分析、改进方向
  - **输出**：结构化反思摘要，存入长期记忆
- **反思提示词框架**：
  ```
  你是一个交易复盘助手，请对以下交易进行反思：
  交易标的：{symbol}
  决策依据：{decision_log}
  实际结果：{profit_loss}
  市场背景：{market_context}
  
  请输出：
  1. 决策正确/错误的关键原因
  2. 哪些假设被验证/证伪
  3. 未来类似场景的改进建议
  4. 可复用的经验模式（一句话总结）
  ```

### 5.6 记忆与辩论的联动

- 辩论启动时：自动加载情景记忆 + 长期记忆中的相关经验
- 辩论进行时：工作记忆实时更新
- 辩论结束后：归档到情景记忆
- 交易执行后：触发反思，存入长期记忆
- 形成**数据飞轮**：使用越多→AI越聪明→用户越满意→使用越多

## 第六章 · Agent 间通信协议（新增章节）

### 6.1 设计原则

- **减少信息损耗**：结构化消息替代自然语言传递，减少歧义
- **可追溯性**：每条消息自带证据链，可回溯到数据源
- **解耦**：异步通信使Agent生命周期独立，支持水平扩展
- **可审计**：所有消息持久化，支持事后复盘

### 6.2 消息格式定义（JSON Schema）

```json
{
  "message_id": "uuid-全局唯一",
  "sender": "agent_name-发送者标识",
  "receiver": "agent_name-接收者或broadcast",
  "message_type": "proposal|challenge|response|vote|command|report",
  "timestamp": "ISO8601-发送时间",
  "session_id": "uuid-所属辩论session",
  "payload": { /* 消息体，按类型定义 */ },
  "confidence": 0.95,
  "evidence_chain": [
    {
      "source": "news_agent",
      "source_message_id": "uuid",
      "summary": "引用摘要"
    }
  ],
  "metadata": {
    "llm_model": "gpt-4o",
    "latency_ms": 1234,
    "token_count": 567
  }
}
```

### 6.3 消息类型定义

| 消息类型 | 发送者 | 接收者 | 用途 |
|---------|-------|-------|------|
| `proposal` | 分析师/大师 | 评审/编排器 | 提交投资方案 |
| `challenge` | 分析师 | 分析师（同组/跨组）| 强制质疑 |
| `response` | 被质疑者 | 质疑者 | 回应质疑 |
| `vote` | 大师/评审 | 编排器 | 决策投票 |
| `command` | 风控Agent | 交易Agent/编排器 | 否决/暂停/执行指令（最高优先级）|
| `report` | 任意Agent | 编排器/用户 | 完整分析报告 |

### 6.4 消息路由机制

| 通信类型 | 协议 | 载体 | 说明 |
|---------|------|------|------|
| 同步消息 | gRPC | 组内Agent直连 | 低延迟，强一致性 |
| 异步消息 | JSON + Kafka | Topic中转 | 解耦，可回溯 |
| 高优先级 | gRPC | 直接通道 | 风控命令、熔断信号 |
| 广播 | Pub/Sub | Kafka | 大师意见分发、状态通知 |

### 6.5 风控消息优先级

风控Agent的 `command` 消息拥有**最高优先级**，交易指令发送前必须经过风控Agent的消息校验。风控消息类型包括：

- `RISK_REJECT`：否决交易（附带否决原因）
- `RISK_WARN`：警告但允许执行（需用户确认）
- `RISK_PAUSE`：暂停所有交易（熔断触发）
- `RISK_REPORT`：每日风控报告

### 12.2 风险控制体系（v1.0 增强版）

#### 12.2.1 风险控制Agent的独立地位（新增）

- **独立于交易决策流程**：不参与方案生成，仅做风险校验
- **直接向用户负责**：不受推荐系统干预，风控报告独立生成
- **一票否决权**：任何交易决策超出风控阈值时直接拦截
- **每日独立风控报告**：推送至用户端

#### 12.2.2 四层风控阈值体系

| 层级 | 监控指标 | 触发动作 |
|------|---------|---------|
| **组合级** | VaR(95%/99%)、最大回撤、贝塔暴露、行业集中度 | 限制新开仓、要求减仓 |
| **个股级** | 单票仓位上限、流动性门槛（日均成交额）| 禁止买入、强制减仓 |
| **执行级** | 频率限制、滑点容忍度、交易时段 | 延迟/拒绝订单 |
| **熔断级** | 市场异常波动、单日亏损达限额 | **暂停所有AI交易，转人工决策** |

#### 12.2.3 风控Agent的否决流程

```
交易信号 → 风控Agent多维度校验 →
  ├── 全部通过 → 进入执行队列
  ├── 风险预警 → 返回预警原因 + 建议调整方向 → 用户确认
  └── 否决 → 返回否决原因 + 建议替代方案 → 重新提案
```

### 12.3 熔断机制（保留 + 增强）

- 市场异常波动（如大盘跌幅>5%）自动暂停AI交易
- 账户单日亏损达用户设定限额立即暂停
- **新增**：风控Agent可自动触发熔断，无需人工介入
- 熔断后转为人工决策模式，用户确认后可恢复

### 13.2 Phase 0：开源基建（第1个月）

目标：建立开源社区，验证核心引擎技术可行性

- [ ] 核心Agent编排框架（Python + LangGraph）— 开源
- [ ] 基础Agent定义：新闻分析Agent / 行情Agent / 教育Agent
- [ ] 结构化通信协议 JSON Schema + 基础实现
- [ ] GitHub 仓库 + README + CI/CD + Contribution Guide
- [ ] 2位大师Agent原型（巴菲特 + 芒格）
- [ ] Docker Compose 一键部署

**关键里程碑**：第一条Issue、第一个外部Pull Request

### 13.3 Phase 1：MVP核心链路（第2-3个月）

目标：跑通核心用户链路，验证产品价值

- [ ] 新闻/行情爬虫 + 同花顺模拟交易连接
- [ ] 新闻Agent升级为深度分析版（多时间窗口+实体抽取）⭐
- [ ] 风险控制Agent独立化 + 一票否决权 ⭐
- [ ] 大师Agent池扩展到5-6位
- [ ] 单引擎回测（事件驱动）
- [ ] 三层记忆系统基础版（工作记忆 + 情景记忆为主）
- [ ] 单模型推荐流程 + 教育Agent "小智" 基础问答
- [ ] Web前端基础版

**关键里程碑**：完成一次完整的"数据→分析→辩论→推荐→交易→反思"闭环

### 13.4 Phase 2：增强版辩论与记忆（第3-5个月）

目标：构筑核心壁垒 —— 深度辩论 + 记忆进化

- [ ] 强制质疑轮 + 跨组交叉质疑完整上线 ⭐
- [ ] 完整的反思机制（交易后自动复盘）
- [ ] 长期记忆向量化 + 语义检索
- [ ] 多引擎回测（蒙特卡洛 + Walk-Forward）⭐
- [ ] Agent通信协议正式版（JSON Schema + Kafka + gRPC）
- [ ] 大师Agent池扩展到8位
- [ ] 用户LoRA微调平台上线
- [ ] Skill/人格市场的MVP

**关键里程碑**：与竞品形成可感知的差异化（辩论质量 + 记忆进化）

### 13.5 Phase 3：商业化与生态（第5-8个月）

目标：构建商业模式 + 开源社区生态

- [ ] 开源社区运营（贡献者激励计划、文档、示例）
- [ ] 企业版开源组件 -> 私有化部署能力
- [ ] 全量大师人格池上线（8-10位）
- [ ] 专业版订阅上线（¥99/月）
- [ ] Skill市场交易系统 + 抽成机制
- [ ] 全引擎回测系统
- [ ] 移动端 Flutter MVP

**关键里程碑**：第一个付费用户、第一个外部商业合作

### 13.6 Phase 4：全球化与进化（第8-12个月）

目标：全球市场 + AI 自我进化

- [ ] 国际数据源覆盖（Yahoo Finance / FinnHub / Reuters）
- [ ] 机构版API开放
- [ ] 强化学习自我博弈优化策略
- [ ] 加密市场覆盖（对标 When.Trade）
- [ ] 多语言支持（英文/日文/东南亚语言）
- [ ] 可视化Agent工作流编辑器

**关键里程碑**：海外用户占比 > 20%
