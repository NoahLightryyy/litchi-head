## 模块定义

交易指令的实际执行落地，对接券商/交易所 API。

**职责边界**：
- ✅ 对接券商/交易所交易接口
- ✅ 订单管理（市价/限价/止损等订单类型）
- ✅ 持仓管理（仓位查询/调整）
- ✅ 交易成本核算（佣金/印花税/滑点）
- ❌ 不负责交易决策生成（那是辩论模块的事）
- ❌ 不负责实时行情获取（那是数据采集模块的事）

## 关键研究问题

- [ ] 实盘接口 vs 模拟接口的抽象：如何让辩论模块不关心底层是模拟还是实盘？
- [ ] 订单状态管理（提交→确认→成交→失败→撤销）的状态机设计？
- [ ] 滑点模型：回测时的滑点假设 vs 实盘的真实滑点？
- [ ] 交易成本：A 股佣金/印花税/过户费怎么算进决策？
- [ ] 接口失败处理：交易所连接断开、订单超时、撤单失败怎么办？
- [ ] 交易日志：每笔交易的全链路追踪（谁决策的、谁批准的、谁执行的、结果如何）？
