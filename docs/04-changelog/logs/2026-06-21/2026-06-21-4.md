### TD-038 密钥安全管理（~30min ✅）

**问题**：`.env` 文件明文存储真实 DEEPSEEK_API_KEY 和 ANTHROPIC_API_KEY，
可能通过 OneDrive 同步、git add --force、截图分享等途径泄露。

**方案**：凭据管理器层（Windows Credential Manager）替代 `.env` 明文存储。

| 文件 | 操作 | 说明 |

| `src/utils/credentials.py` | 🆕 | 凭据管理器模块 — keyring 封装，优先级链：系统凭据管理器→环境变量→.env |
| `src/utils/config.py` | 📝 | Settings.model_post_init 从凭据管理器覆盖 API Keys，`get_key_source()` 诊断方法 |
| `scripts/store-api-keys.py` | 🆕 | 交互式密钥存储 CLI（--status/--set/--delete） |
| `pyproject.toml` | 📝 | 添加 `keyring>=25.0` 依赖 |
| `.env` | 📝 | 清空明文密钥，改为凭据管理器占位符 |
| `.env.example` | 📝 | 更新注释说明新流程 |

#### 密钥迁移

- 3 个密钥已从 `.env` 迁移到 Windows Credential Manager（服务名: `litchi-head`）
- 迁移后 `.env` 仅保留空占位符
- 密钥读取优先级：Credential Manager → 环境变量 → `.env` fallback
