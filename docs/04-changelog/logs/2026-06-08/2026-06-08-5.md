# AI 工作日志 — 2026-06-08 (第 8 次)

> 本次会话：修复 DeepSeek API 401 — 环境变量端点混用

---

## 问题

Claude Code 主会话显示 `deepseek-chat · API Usage Billing`，但持续报 `401 API Key 无效`，重试 10 次无响应。

**根因**：全局 `ANTHROPIC_BASE_URL` 被设为灵算（`lingsuan.top`），主会话却把 DeepSeek Key 发到灵算端点 → 401。两个 DeepSeek Key 实测均有效。

---

## 本次修复

| 配置层 | 改前 | 改后 |
|--------|------|------|
| Windows `ANTHROPIC_BASE_URL` | `lingsuan.top` | `https://api.deepseek.com/anthropic` |
| Windows `ANTHROPIC_AUTH_TOKEN` | DeepSeek Key（但 BASE_URL 错） | DeepSeek Key（与 `DEEPSEEK_API_KEY` 一致） |
| `~/.bashrc` | 灵算 BASE_URL + `AUTH_TOKEN=$ANTHROPIC_API_KEY` | DeepSeek BASE_URL + `AUTH_TOKEN=$DEEPSEEK_API_KEY` |
| `.env` `DEEPSEEK_API_KEY` | 与 Windows 不一致的另一把 Key | 与 Windows 统一 |
| `.env` 灵算配置 | 仅有 `AUTH_TOKEN` | 补全 `ANTHROPIC_API_KEY`，保留灵算给 Python `provider=anthropic` |
| `CLAUDE.md` | 误导：全局写灵算 | 区分 Claude Code 主会话 vs `.env` Python 代码 |
| `.env.example` | 注释不清 | 标明两套 Key 用途 |

---

## 验证

- DeepSeek direct API：200 OK
- DeepSeek anthropic-compat（`api.deepseek.com/anthropic`）：200 OK
- DeepSeek Key → lingsuan：401（复现原问题）

---

## 架构结论

Claude Code 的 `ANTHROPIC_BASE_URL` 是会话级，**同一进程无法**「主会话 DeepSeek + 子 Agent 灵算 Claude」。灵算 Key 仅用于项目 `.env` 中 Python 代码显式 `provider="anthropic"`。

---

## 用户操作

1. **完全退出** Claude Code 进程
2. **重新启动** Claude Code
3. 建议在灵算后台**轮换**已泄露的 API Key
