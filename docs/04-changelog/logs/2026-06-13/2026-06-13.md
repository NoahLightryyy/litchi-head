### 🥇 Phase 1: 分析师层实施

**新增 `src/debate/analysts.py`**：
- `AnalystPersona(BaseModel)` — 分析师人格定义（专注领域 + 方法论 + 输出要求）
- 4 位内置分析师：

| analyst_type | 名称 | 专注领域 |
|:---|:---|:---|
| `fundamental` | 基本面分析师 | ROE/PE/财务/估值/护城河 |
| `technical` | 技术面分析师 | 趋势/支撑阻力/RSI/MACD/成交量 |
| `sentiment` | 情绪面分析师 | 新闻舆情/市场关注度/预期差 |
| `macro` | 宏观面分析师 | 利率/经济周期/行业趋势/政策 |

**新增 `AnalystReport` 模型**（`src/debate/models.py`）：
- `analyst_type` / `key_findings` / `data_evidence` / `red_flags` / `confidence` / `summary` / `score` / `direction_hint` / `latency_ms`

**新增 `analyst_round` 节点**（`src/debate/orchestrator.py`）：
- `make_analyst_round_node()` — 顺序运行 4 位分析师
- `_run_single_analyst()` — 调用 `llm_service.invoke_structured(AnalystReport)`，失败返回默认值
- `_format_analyst_reports()` — 将 4 份报告格式化为策略师 readable 的 prompt

**策略层重构**（原 `master_round`）：
- `_run_single_master()` 新增 `analyst_reports` 参数
- Prompt 构建改为：分析师报告（主要输入）→ 市场数据（辅助）→ 战略合成指示 → 历史上下文
- `make_master_round_node()` 从 state 读取 `analyst_reports` 并传递

**图结构变更**：
```
旧：collect_data → master_round → review_round → review_report → aggregate
新：collect_data → analyst_round → strategy_round(master) → review_round → review_report → aggregate
```

**向后兼容**：`DebateOrchestrator.__init__()` 新增可选 `analyst_personas` 参数，默认 `get_default_analysts()`
