# 架构批判与市场对照

> 2026-06-14 架构审视 — 将 litchi-head 的 9 层专业交易团队架构
> 与市场三大路线（TradingAgents / AI Hedge Fund / ContestTrade / EvoTraders）做
> 逐层对照和批判性评估。**目的不是自我否定，而是看清差距、排准优先级。**

---

## 一、市场格局：四条技术路线

| 路线 | 代表项目 | 核心理念 | 成熟度 |
|:-----|:---------|:---------|:------:|
| **角色分工线** | TradingAgents (71k⭐) | 模拟真实投行团队：分析师→研究员→交易员→风控→PM | 🟢 最高 |
| **人格模拟线** | AI Hedge Fund (53k⭐) | 14 位投资大师人格独立判断 + Portfolio Manager 综合 | 🟢 成熟 |
| **内部竞赛线** | ContestTrade (新兴) | 多 Agent 互相竞争、实时排名、优胜劣汰 | 🟡 论文→产品 |
| **自进化线** | EvoTraders/ReMe (阿里) | 反思记忆 + 历史轨迹学习 + 跨交易日行为进化 | 🟡 论文→产品 |

**litchi-head 属于角色分工线**，与 TradingAgents 直接对标。但在角色分工线的框架内，**融合了人格模拟线的大师人格**（策略师层），这是差异化。

## 三、优势：做得比竞品好的地方

### 3.1 🏆 分析师→策略师 双层设计（市场独有）

TradingAgents 的分析师直接产出给 Bull/Bear 研究员，中间没有策略综合层。litchi-head 插入了一层「策略师」——分析师产出**纯事实报告**（key_findings/red_flags），策略师带着大师人格做**主观综合判断**。

```
TradingAgents:  分析师(raw data) ──→ 研究员(bull/bear take)
litchi-head:    分析师(facts only) ──→ 策略师(opinion + personality) ──→ 辩论
                                    ↑
                          "事实"与"判断"解耦
```

这对应真实投研流程中「研究员写报告 → 基金经理做判断」的分工。竞品都跳过了这一步——这是 litchi-head 最亮眼的架构决策。

**保持策略**：不合并。如果为了减少层数去合并这两个，会损失"事实/判断分离"这个核心差异化。

### 3.2 🏆 D3 独立评审层（市场独有）

TradingAgents 的辩论胜负由 Research Manager 一人判断——**辩论者和裁判者是同一个人**。litchi-head 在辩论之外设置独立的裁判 Agent（D3），裁判不参与辩论、只看辩论记录做评分和权重建议。

这是司法体系中「陪审团 + 法官分离」的思想。ContestTrade 用竞赛机制达到类似效果（市场反馈是最公正的裁判），但没有独立的评审 Agent。

**保持策略**：不弱化。D3 的独立性是质量的保证。后续可以通过 M3（信任度评分）数据化 D3 的裁判质量。

### 3.3 🏆 M1 历史注入 + 全链路记忆

TradingAgents 的记忆只注入到决策阶段。litchi-head 的记忆层贯穿全链路——分析师能看到历史决策、辩论时能引用上次结果、风控能对比历史风控判断。

**保持策略**：继续强化。但需要补上"记忆→反思→进化"的闭环（见 §4.1）。

## 四、劣势：硬伤和必须解决的差距

### 4.1 🔴 致命缺陷：没有反思/学习闭环

**症状**：9 层链路到 PM 裁决就结束了。裁决之后呢？等三个月看实际收益？还是永远不知道自己的决策对不对？

**竞品怎么做**：

| 项目 | 反思机制 | 效果 |
|:-----|:---------|:-----|
| TradingAgents | Decision Memory：记录决策→等实际收益→对比→反思→注入下次 | 轻量但有效 |
| EvoTraders/ReMe | 每日交易后自动计算 P&L → 多维分析 → 存储轨迹 → 向量检索复用 | 任务成功率 +15pp |
| ContestTrade | 真实市场反馈持续排位 Agent → 长期不准的降权/淘汰 | 累计收益 52.8% vs 对照组 -19.1% |

**litchi-head 现状**：零。9 层分析→决策→输出，然后没了。

**影响**：
- 系统永远是"第一次分析"，不会越做越好
- 错误模式会重复——因为没有记录和反思
- 无法回答"这个系统过去判断得怎么样"——用户凭什么信任？
- 这是与竞品之间 **最大的结构性差距**

**修复方向**（M2）：
1. **轻量起步**：TradingAgents 的 Decision Memory 模式——记录决策 + 实际收益出来后对比 + 生成反思文本 + 注入下次 prompt
2. **中期升级**：EvoTraders 的 ReMe 模式——完整执行轨迹 + 向量检索 + 长短期记忆分离
3. **最终形态**：ContestTrade 的排位模式——每个 Agent 有历史准确率追踪 + 权重动态调整（M3）

### 4.2 🔴 层数膨胀的成本风险

**症状**：一个 ticker 跑一遍 = **至少 15 次 LLM 调用**（4 分析师 + 5 策略师 + 辩论 round + 评审 + 3 风控 + 1 交易员 + 1 PM）。

TradingAgents 约 8-10 次调用，litchi-head 是它的 1.5-2 倍。

**竞品怎么做**：

| 项目 | 成本控制 |
|:-----|:---------|
| TradingAgents | Fast/Slow 双模路由：简单任务用 cheap model，复杂任务用 powerful model |
| AI Hedge Fund | 支持 --ollama 本地模型，零 API 成本 |
| ContestTrade | 竞赛机制淘汰低效 Agent，减少无效调用 |

**litchi-head 现状**：有 `complexity_router.py`，但只在 LLM 调用层做模型选择，没有在编排层做「哪些节点可以跳过、哪些可以合并」。

**修复方向**：
1. **短路优化**：如果数据层返回空（无行情/停牌）→ 直接结束，不跑后续 8 层
2. **层合并**：对于简单问题（"这个股票值不值得看"），可以跳过分析师层、直接策略师层读数据
3. **模型分层**：分析师层用 cheap model（deepseek-chat），策略师/辩论/风控用 reasoner

### 4.3 🟡 缺少 ContestTrade 式的 Agent 质量追踪

**症状**：D4 的权重是 LLM 给的，不是数据驱动的。巴菲特策略师和利弗莫尔策略师，谁的历史判断更准？不知道。

**竞品怎么做**：
- ContestTrade：每个 Agent 有历史得分 + 实时排名 + 长期不准的降权/淘汰
- TradingAgents：Decision Memory 会对比预测 vs 实际，有方向判断准确率

**修复方向**（M3）：信任度评分系统——每个 Agent 的输出 vs 实际市场结果，累积追踪准确率，动态调整其在 D4 聚合中的权重。

### 4.4 🟡 强依赖单一 LLM 提供商

TradingAgents 支持 7+ 种 LLM（OpenAI/Anthropic/DeepSeek/Gemini/Grok/Qwen/GLM + Ollama）。litchi-head 的 LLMService 架构上支持多 provider，但实际只在 DeepSeek 上验证过。

**修复方向**：优先级低（DeepSeek 当前稳定），但应在 Phase 2 末期补充至少一个 fallback provider（如 Ollama 本地模型）的集成测试。

## 五、与竞品的差异化总结

### 5.1 产品定位差异

| 维度 | TradingAgents | AI Hedge Fund | litchi-head |
|:-----|:-------------|:------------|:-----------|
| **目标用户** | 量化爱好者/研究者 | 散户/独立投资者 | **散户/新人（中国 A 股）** |
| **哲学** | 模拟华尔街机构流程 | 让大师帮你"看一看" | **15 秒出结果，10 秒能看懂** |
| **交互方式** | CLI 命令行 | Web UI + 可视化 | 一句话输入（计划中） |
| **决策深度** | 5 层递进 | 19 人投票 | **9 层递进 + 双层分析** |
| **可解释性** | ReAct 推理链 | 大师观点罗列 | **结构化 Pydantic 输出** |
| **中国市场** | ❌（仅 astock 分支） | ❌ | ✅ **主战场** |

### 5.2 litchi-head 的生存空间

litchi-head 不是 TradingAgents 的竞争者——是**互补者**：

- TradingAgents 做的是"通用投研流水线"（全球市场、学术研究）
- litchi-head 做的是"中国 A 股散户友好型决策工具"（一句话→15 秒→能行动）

这个定位在 A 股市场上暂无直接竞品——ContestTrade 也是 A 股但走竞赛路线，且是商业产品不开源。

## 六、基于市场对照的优先级重排

| 原优先级 | 新建议 | 代号 | 理由 |
|:------:|:------:|:-----|:-----|
| 📌 | 🔴 **提升为第一优先** | **M2 交易后反思** | 与竞品最大结构性差距，缺少它就是"只说不练" |
| 🥈 | 🟡 保持 | 端到端链路验证 | 验证 9 层实际跑通（会暴露成本问题） |
| 🥈 | 🔴 **提升** | **回测引擎基础** | 没有回测就无法验证决策质量，M2 反思也无从谈起 |
| 🥈 | 🟡 保持（M2 前置依赖） | **M3 信任度评分** | M2 反思需要 M3 的评分数据作为输入 |
| 🥈 | ⬇️ 降级 | C1 简报分区 | 纯展示优化，不改变决策质量 |
