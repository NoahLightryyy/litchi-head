## 所属战线：全流程量化交易框架（回测子层）

> 回测不属于独立战线，而是**全流程量化交易战线的验证环节**。但值得单独拎出来是因为"LLM 辩论的回测"和"传统量化回测"有本质不同。
>
> [查看战线完整分析](../../行业开源方案对照知识库.md#-战线-3全流程量化交易框架)

### 在这个战场中的角色

```
全流程量化交易：
  数据 → 分析/策略 → 回测验证 → 优化 → 实盘
                          ↑
                   你在这里（未来）

传统量化的回测：
  写策略代码 → 加载历史数据 → 逐Bar回放 → 算绩效指标
  （策略逻辑是确定性代码）

LLM辩论的回测（你的情况）：
  读历史行情+新闻 → LLM辩论决策 → 记录决策 → 和真实后续走势对比
  （策略逻辑是LLM输出，每轮回测结果可能不同）
```

**核心认识**：你的回测和传统量化回测完全不同——传统回测是**确定性**的（同一代码跑同一段数据结果一样），你的回测是**概率性**的（LLM 每次输出可能不同）。这意味着你需要自己设计回测方案，不能直接用 Backtrader 之类的工具。

**职责边界**：
- ✅ 历史数据上的策略回测
- ✅ 绩效指标计算（收益率/夏普/最大回撤等）

- ❌ 不负责因子有效性验证（那是因子研究模块的事）

## 回测的特殊挑战（LLM 辩论 vs 传统策略）

| 维度 | 传统量化回测 | 你的 LLM 辩论回测 |
|------|------------|-----------------|
| **确定性** | 同一代码+同一数据 = 同一结果 | 每次 LLM 输出可能不同 → 需多次运行取统计结果 |
| **执行速度** | 毫秒级跑完全部历史 | 每次决策需 LLM 调用（秒级）→ 无法细粒度回放 |
| **数据需求** | 只需要 K 线数据 | 需要 K 线 + 新闻 + 财报 + 舆情（回测时需要"还原当时的上下文"）|
| **前视偏差** | 容易控制（不泄露未来数据） | 容易翻车（LLM 训练数据可能已经"见过"这段历史）|
| **评估指标** | 收益率/夏普/最大回撤等 | 除了这些，还要看"决策一致性"、"框架泛化能力" |

## 关键研究问题

### 设计层
- [ ] 你的"虚拟投资"功能：如何在不让 Agent 知道"未来走势"的前提下回放历史？
- [ ] "时光机"压力测试：如何切分时间窗口（如"2024年春节后行情"、"2022年4月大跌"）？
- [ ] 多次回测取统计值：同一段历史跑 10 次 Agent，结果不一致怎么办？

### 评估层
- [ ] 你的回测报告应该包含哪些指标？（收益率/夏普/最大回撤/胜率/决策一致性……）
- [ ] 如何评估"决策质量"而不仅仅是"赚钱了吗"？（比如：逻辑严密度、风险意识、信息利用效率）

### 技术层
- [ ] 回测时如何避免 LLM 的前视偏差？（LLM 训练数据已经包含历史新闻）
- [ ] LLM 一次决策需要 3-8 秒，一年的日线数据就是 40 分钟，性能优化方案？
