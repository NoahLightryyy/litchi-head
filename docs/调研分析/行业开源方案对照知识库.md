### 战线 1：多 Agent 辩论决策 ⬅ 你的主战场

**格局速览：三条路线在竞争**

| 路线 | 代表 | 核心理念 |
|------|------|---------|
| **角色分工线** | TradingAgents | 模拟真实投研团队：分析师→研究员→交易员→风控→PM |
| **人格模拟线** | AI Hedge Fund | 投资大师人格（巴菲特/芒格等）各自独立分析再综合 |
| **内部竞赛线** | ContestTrade | 多个Agent互相竞争，优胜劣汰，只采纳最优者 |

#### 🥇 TradingAgents — 目前这条战线的领头羊

**谁在做**：TauricResearch（作者肖易佳，清华→UCLA，唐杰教授学生）
**Stars**：71k+（整个AI交易领域最高）
**论文**：arXiv 2412.20138

**五层结构（这是你的直接参照对象）：**

```
分析师层（并行）                   研究员层（辩论）              交易员+风控+PM
┌──────────────┐              ┌─────────────────┐         ┌─────────────────┐
│ 基本面 Analyst│──并行分析──→  │  Bull Researcher│──对抗──→│ Trader(交易员)  │
│ 舆情  Analyst│──并行分析──→  │ Bear Researcher │──反驳──→│ Risk(3维风控)    │
│ 新闻  Analyst│──并行分析──→  │ Research Mgr    │──综合──→│ Portfolio Mgr   │
│ 技术  Analyst│──并行分析──→  └─────────────────┘         └─────────────────┘
└──────────────┘                    ↑                          │
                               多轮辩论(默认2轮)      最终决策Buy/Hold/Sell
```

**关键设计（值得你仔细看源码的）：**
1. **强制立场辩论** — Bull/Bear 被强制背对背写分析，不是自由讨论，避免"两头堵"
2. **多轮对抗** — 读完对方观点后必须反驳，逼出更深层推理
3. **分层风控** — 激进/保守/中性三维度，不是单一点评
4. **Decision Memory** — 记录历史决策+实际收益，下次自动注入参考
5. **Checkpoint/Resume** — SQLite 存中间状态，崩了继续

**工程实现**：
- LangGraph StateGraph 编排
- Pydantic schema 结构化输出（v0.2.4）
- 支持 10+ LLM Provider（OpenAI/Claude/DeepSeek/Qwen/GLM/Ollama）
- 成本：单只股票约 $0.30-$0.50

#### 🥈 AI Hedge Fund — 人格模拟线代表

**谁在做**：Virat Singh
**Stars**：51k+
**核心差异**：不是"一个分析系统"，而是"14位传奇投资人各抒己见"

```
14位大师人格（各有一套投资哲学提示词）       5个分析Agent（并行）
┌───────────────────┐                  ┌──────────────────────┐
│ Warren Buffett    │ → 价值投资视角    │ Valuation Agent     │
│ Ben Graham        │ → 安全边际视角    │ Sentiment Agent     │
│ Michael Burry     │ → 做空/逆向视角   │ Fundamentals Agent  │
│ Cathie Wood       │ → 颠覆性增长视角  │ Technicals Agent    │
│ Nassim Taleb      │ → 尾部风险视角    │ Risk Agent          │
│ Charlie Munger    │ → 多元思维视角    └──────────────────────┘
```

**不足之处**：
- 没有真正的辩论/对抗机制，只是各自投票
- 没有记忆/反思系统
- 不执行交易，只出分析建议

#### 🥉 ContestTrade — 内部竞赛线（最接近你的四组辩论理念）

**谁在做**：财跃星辰（界面财联社旗下），上海开源典型案例
**论文**：arXiv 2508.00554
**核心创新**：不是Agent合作，而是Agent竞争

```
数据团队（并行）                    内部竞赛                   研究团队（并行）
┌──────────────┐              ┌──────────────┐            ┌──────────────┐
│ Data Agent 1 │──生成因子──→  │ 评分+排名    │            │ Researcher 1│
│ Data Agent 2 │──生成因子──→  │ 只采纳Top   │──最优因子→  │ Researcher 2│
│ Data Agent 3 │──生成因子──→  │             │──注入信念→  │ Researcher 3│
└──────────────┘              └──────────────┘            └──────────────┘
                                                                ↓
                                                          第二轮内部竞赛
                                                                ↓
                                                          最终资产配置
```

**和你的异同**：
- 你也用多组辩论，但你是"讨论→综合"，ContestTrade 是"竞争→优胜劣汰"
- 你的评审Agent是"裁判"，他们的竞赛机制是"淘汰赛"
- 可以结合：先竞赛筛选 → 再辩论深入 → 最后由评审综合

**中国市场优势**：支持 Tushare、Bocha、AKShare，适配A股

### 战线 2：AI 量化研究平台

**格局：微软一家独大，Qlib + RD-Agent 双引擎**

#### Qlib — AI 量化研究的事实标准

**GitHub**：microsoft/qlib · 36k+⭐
**定位**：AI 导向的量化投资平台，从因子挖掘到回测全流程

**架构（数据→特征→模型→组合→回测全管线）：**

```
原始数据 → DataHandler → 特征工程(Alpha158/360) → 模型训练 → 预测
    ↓
组合策略(TopK/EnhancedIndexing) → 回测执行 → 绩效分析
```

**内置模型**：30+ SOTA（LightGBM / XGBoost / LSTM / Transformer / GATs / TabNet...）
**数据加载速度**：800只股票14年数据+14因子 → 7.4秒（MySQL方案要6分钟+）

#### RD-Agent — 自进化因子研究（2026年最重磅更新）

**GitHub**：microsoft/RD-Agent · 12k+
**论文**：NeurIPS 2025

**核心：R&D 双循环**

```
R-Agent（研究）                      D-Agent（开发）
┌──────────────┐              ┌────────────────────┐
│ 假设生成      │──实验设计──→  │ 代码生成(CoSTEER)  │
│ 文献综述      │              │ 数据获取/清洗      │
│ 结果解读      │←──实验报告──│ 特征工程/模型训练   │
└──────────────┘              │ 回测验证            │
        ↑                      └────────────────────┘
        │                               ↑
        └────────── 反馈分析 ────────────┘
```

**实测结果**（国联民生证券 2026.5）：
- 36轮 Loop，IC 提升至 0.07
- 11次 SOTA 更新
- 预估有效因子产出约 900 个/月
- 每轮成本 < 10美元

**五大单元**：规范(Spec)→构思(Synthesis)→实现(CoSTEER)→验证(Runner)→分析(Summarizer)

#### 和你项目的关系

Qlib/RD-Agent 做的是**AI因子研究**，你做的是**多Agent辩论决策**，这两条线是互补的：
- Qlib 回答"什么因子有效"
- 你的系统回答"这些因子/数据综合起来该不该买"
- 未来 Phase 2+ 可以在你的 pipeline 中集成 Qlib 的因子分析结果

### 战线 3：全流程量化交易框架

**格局：vnpy 守中国，QuantConnect 攻全球**

#### vnpy（VeighNa）— 你的"未来实盘对接"参照对象

**GitHub**：vnpy/vnpy · 37k+⭐
**定位**：中国量化交易框架的标准答案
**语言**：Python + Rust（VnpyRS）

**为什么你必须关注 vnpy**：
- 你未来的实盘交易几乎避不开它（CTP/中金所/恒生接口都在它这）
- 它的事件驱动引擎是生产级设计标准
- 40+ 交易所API接口，中国最全

**架构（事件驱动 + 插件式）：**

```
【事件引擎 EventEngine】 ← 行情/订单/成交事件
     ↓
【主引擎 MainEngine】 → 策略模块（CTA套利/价差/期权）
     ↓
【数据层】 → 数据库适配（SQL/MongoDB/DolphinDB/InfluxDB）
     ↓
【接口层】 → CTP / XTP / 币安 / IB 等40+接口
```

**2026 核心更新**：
- Rust 重构 VnpyRS → 单策略延迟 < 200μs（提升85%）
- vnpy.alpha 集成 Alpha158 因子库 + LightGBM/LSTM/MLP
- 分布式部署（RPC 多进程协作）

**vnpy 4.0 新增值得关注的**：
- `vnpy.alpha` — AI 引擎模块，你的模型可以挂在这里
- 事件驱动模式 → 你未来多Agent接收实时行情时可能会用到

#### QuantConnect / LEAN — 全球化参考

**定位**：云端量化工厂，130+市场，C#核心
**优势**：数据质量最高（400TB无存活者偏差）、多资产类别
**劣势**：A股基本不支持、云端依赖、C#语言门槛
**商业化**：专业版约 $50/月起

#### ⚠️ Backtrader — 别用，纯学习

**Stars**：15k+，但2019年后已停更
**Python 3.10+ 兼容性问题**，不适合新项目
**唯一价值**：代码极其简洁，适合理解事件驱动回测的原理

#### OpenBB — "开源 Bloomberg"

**GitHub**：OpenBB-finance/OpenBB · 64k+⭐
**定位**：Connect once, consume everywhere

**分层架构（你的数据层可参照这个）：**

```
┌──────────────────────────────────────────────┐
│  接口层：CLI / Python SDK / REST API / MCP   │
├──────────────────────────────────────────────┤
│  核心平台层：Router / CommandRunner /         │
│             ExtensionLoader / ProviderInterface│
├──────────────────────────────────────────────┤
│  Provider系统（每个数据源独立实现）            │
│  ├─ YahooFinanceProvider                      │
│  ├─ FREDProvider（美联储数据）                │
│  ├─ AKShareProvider（A股数据）                │
│  └─ 100+ 其他 Provider                        │
├──────────────────────────────────────────────┤
│  数据模型层：Pydantic 强类型                  │
└──────────────────────────────────────────────┘
```

**每个 Provider 的 Fetcher 三方法模式（值得你抄作业）：**

```python
class AkShareFetcher(Fetcher):
    def transform_query(self, params: EquityQuoteParams) -> AkShareQuery:
        """标准化参数 → Provider 特有参数"""
        
    def extract_data(self, query: AkShareQuery) -> pd.DataFrame:
        """实际 API 调用（调 akshare 函数）"""
        
    def transform_data(self, raw: pd.DataFrame) -> EquityQuoteData:
        """原始 DataFrame → Pydantic 模型"""
```

**2026关键更新**：
- MCP Server 支持 → AI 代理可直接查金融数据
- Snowflake Native App → 企业级部署
- OpenBB Workspace → Web UI 拖拽分析

#### AKShare — 你已经在用的

A股数据源的事实标准，但注意：
- **接口不稳定**——上游网站改版会直接影响（你已经有缓存层，还要加适配器层）
- **无统一抽象层**——每个函数签名不同，不像 OpenBB 那样做成统一 Provider

#### 💡 对你的启示

你的 `src/data/collector.py` 目前直接调 akshare 函数，可以借鉴 OpenBB 的 Provider 模式：
1. 定义 `DataProvider` 抽象接口
2. 实现 `AkShareProvider`、`TuShareProvider`、未来自定义数据源
3. 上层代码不关心数据源哪来的

### 战线 5：自进化/反思系统

**格局：两个方向——EvoTraders 的 ReMe 记忆系统 vs TradingAgents 的 Decision Memory**

#### EvoTraders (ReMe) — 最完整的自进化方案

**谁在做**：阿里巴巴通义实验室 · AgentScope 团队
**论文**：ArXiv 2512.10696

**每日交易六阶段（含反思环节）：**

```
Phase 0: 清空短期记忆（准备新一天）
Phase 1: 4位分析师独立分析 + 工具调用
Phase 2: 智能体间会议讨论（MsgHub 群聊模式）
Phase 3: 投资组合经理做最终决策
Phase 4: 执行交易
Phase 5: ─→ 反思阶段 ←─ 这是核心
              ├─ 计算实际盈亏
              ├─ 分析对错原因
              └─ 存入 ReMe 长期记忆 → 影响未来决策
```

**ReMe 三阶段循环**：
1. **经验提取** → 多维分析执行轨迹（成功/失败/对比），经 LLM-as-a-Judge 验证
2. **上下文适配重用** → 场景感知索引，不是简单基于查询匹配
3. **效用驱动精炼** → 主动加有效记忆、删过时内容

**实测效果**：Qwen3-32B + ReMe 任务成功率 56.05%，较无记忆基线提升 15pp

#### TradingAgents Decision Memory — 更轻量的方案

- 用 `trading_memory.md` 纯文本记录所有历史决策+实际收益
- 下次运行时自动注入 prompt 作为参考
- 实现极简单，但效果明显
- 缺点：记忆量增长会撑爆 context

#### 💡 对你的启示

你的 `src/memory/store.py` 目前是通用 key-value 存储，还没有"反思→进化"闭环：

```
建议路线：
Phase 1（当前）：JSON key-value 存储 ✓
Phase 2（下一步）：增加 反思触发器（交易后自动分析对错）
Phase 3（未来）：升级到 向量数据库 + 语义检索
```

可学习 EvoTraders 的 ReMe 三阶段，但初期先用 TradingAgents 的轻量方式（纯文本记忆文件）跑通流程。
