# 多智能体投资决策平台产品设计方案

## 一、产品愿景与目标

打造一个融合**多智能体协同**、**用户共创**与**自动化交易**的下一代投资辅助平台。通过多个专业AI Agent协作采集新闻、行情，连接同花顺实盘/模拟账户，并在用户可控的前提下提供深度投研、智能托管与社区化策略分享。让专业级量化分析与主观交易经验，以极低的使用门槛触达每一位投资者。

---

## 二、系统整体架构

平台采用**云原生微服务 + 多Agent协同引擎**设计，核心分为六层：

| 层级 | 说明 | 关键组件 |
|------|------|----------|
| **接入层** | 用户交互 | Web前端、移动端、同花顺客户端插件 |
| **API网关** | 统一鉴权、路由、限流 | Kong / APISIX，集成OAuth2.0 |
| **应用服务层** | 业务功能 | 推荐引擎、辩论工厂、社区服务、教育助手、回测服务 |
| **Agent编排层** | 多模型工作流 | 任务调度器、多Agent对话管理、辩论流程控制器、记忆模块 |
| **数据与模型层** | 持久化与模型推理 | 新闻/行情数据库、向量知识库、模型池（多个LLM API + 微调实例） |
| **基础设施层** | 算力与采集 | Kubernetes集群、爬虫集群、消息队列(Kafka)、Redis缓存 |

**多Agent工作流**的核心运行逻辑：

1. 事件触发（定时/用户请求/行情异动）→ 2. 调度器并行拉起新闻采集Agent、市场数据Agent，并确认同花顺账户Agent连接状态  
2. 数据就绪后，启动**四组辩论Agent群**，并行产出四份分析报告与交易方案  
3. 辩论结果汇总，交由融合排序服务，结合用户风险偏好与资金生成最终推荐  
4. 若开启托管，交易决策发送至同花顺执行Agent，执行结果返回并记录日志，用于后续强化学习微调

## 三、Agent角色定义与工作流

### 3.1 基础Agent角色

| Agent | 职责 | 技术实现 |
|-------|------|----------|
| **新闻采集Agent** | 定时爬取财联社、雪球、东方财富等新闻，进行实体识别、情感打分、摘要提取 | Scrapy/Playwright + NLP情感分析模型 |
| **行情采集Agent** | 连接股市数据源（新浪/聚宽/交易所Level2），获取实时报价、K线、技术指标、板块资金流 | Websocket行情网关 + 时序数据库 |
| **同花顺账户Agent** | 安全桥接用户同花顺账户，查询持仓/可用资金，执行买卖指令，返回成交信息 | 同花顺OpenAPI / 模拟交易协议，金融安全沙箱 |
| **教育助手Agent** | 基于大模型+RAG，解释MACD、KDJ、PE等指标，梳理行业上下游与热点板块 | LangChain + 向量库（指标、行业研报） |
| **交易策略Agent** | 依据用户配置的skill/人格，或系统辩论结果，生成最终交易指令 | 策略运行时，连接回测引擎和执行层 |

### 3.2 核心辩论工作流（功能三深度实现）

在生成投资推荐时，启动“四组辩论评审”机制，每组包含1个评审模型和3个分析师模型，按如下流程运行：

```
输入: 当前持仓、资金、偏好(激进/保守)、新闻聚合、行情快照

并行启动四组 (代表四种投资哲学):
  [价值组]   [成长组]   [趋势组]   [量化对冲组]

每组内部流程（多轮对话）:
  1. 三个分析师模型分别从不同维度提出初始方案（如重仓/轻仓/对冲）
  2. 进入“吵架”轮：互相质疑对方的风险、收益逻辑，寻找漏洞
  3. 评审模型总结讨论，消除矛盾，提炼出一套融合方案
  4. 评审模型对该方案进行评分（1-10），并提取三个核心关键词

四组全部完成后:
  - 综合排序服务根据评分、用户偏好、回撤预期等，排出最终推荐方案列表
  - 同时展示各组关键词云，让用户一眼看懂分歧点
  - 用户可选择某一组方案执行，或授权系统自动选择最优
```

示例提示词框架（评审模型）：
```
你是一个资深投资评审员，负责评估三位分析师的辩论。
辩论记录：{debate_log}
请综合他们的观点，输出：
1. 最终投资组合方案（股票代码、仓位、理由）
2. 综合评分（1-10）
3. 三个关键词（如“高安全边际”“政策驱动”“超跌反弹”）
```

系统同时会记录每一次辩论的胜出方案与真实市场结果，构成“群体智能进化”的数据基础。

## 四、各功能详细设计

### 功能一：智能推荐与托管交易

**用户流程**：
1. 绑定同花顺账户（实盘/模拟），系统读取总资产、持仓、历史盈亏。
2. 在“偏好卡”中拖动滑块：保守（0）↔ 激进（100），也可选择“持有现金为主”、“定投模式”等标签。
3. 点击“生成今日推荐”，触发多Agent工作流（新闻+行情采集 → 四组辩论 → 综合推荐）。
4. 推荐结果以卡片展示每组的方案、评分、关键词和模拟回测收益曲线（基于未来数据的模拟？实际用历史相似场景回测）。
5. **托管开关**：用户设定单日最大亏损、单票仓位上限后，开启托管，系统自动执行交易。每次交易前会推送通知，用户可在15分钟内“取消”，否则自动下单。

**风控层**：
- 动态止盈止损线由AI根据波动率实时计算。
- 账户Agent内部集成验资、验券、频率限制，符合交易所规则。
- 所有托管操作均记录在不可篡改的日志中。

### 功能二：用户友好的模型微调参与

让普通用户像训练“游戏角色”一样训练自己的交易AI：

**“AI交易人格训练营”向导**：
1. **基础模型选择**：展示三到四个不同性格的基础模型（“谨慎的乌龟”“敏锐的猎豹”“均衡的章鱼”），配有语言描述和过往风格回测曲线。
2. **风格调整**：通过拖拽条或自然语言输入——“我希望它更喜欢科技股，但单只股票不要超过20%仓位”“遇到大跌要冷静抄底”。系统将文本转为偏好约束。
3. **反馈教学**：用户对每日推荐方案进行👍/👎，或直接手动修改订单。这些反馈形成偏好数据集，在夜间以**参数高效微调（LoRA）**方式更新用户专属模型分支。用户看到“训练进度条”和“模拟成绩变化趋势”。
4. **评估与发布**：微调完成后，自动在历史行情中进行滚动回测，展示夏普比率、最大回撤等改善幅度。用户满意后，可一键设为“我的主力交易AI”，用于托管交易。

整个过程用户无感接触代码，仅需提供偏好与反馈。

### 功能三：四组辩论评审与方案对比

已在上文3.2详细描述。前端呈现上将引入**多方案对比视图**：

- 四个选项卡分别展示各组的方案、持仓饼图、核心关键词。
- 支持雷达图比较：盈利能力、风险控制、流动性、分散度、新闻敏感度等维度。
- 用户可点击某个组“采用此方案”立即下单，或勾选“融合”让系统再调用一个综合模型合并优点生成最终计划。

### 功能四：社区化Skill与人格市场

**Skill定义**：一个Skill是一套可复用的交易逻辑包，包含：
- 选股因子/规则（如“连续3日放量+突破年线”）
- 择时信号（基于MACD金叉或新闻情绪转折）
- 仓位管理算法（固定比例、凯利公式、动态再平衡）
- 元数据：作者、描述、标签、历史回测绩效图。

**上传与分享**：
- 提供低代码Skill构建器：拖拽条件、添加因子，或直接上传Python策略代码（沙箱执行）。
- 社区中每个Skill展示标准化回测结果（近1年、3年收益/回撤），确保公平对比。
- 排行榜机制：按风险调整后收益、订阅数、跟单盈利额排名。

**人格训练与交易**：
- 平台提供“大师人格包”：收集巴菲特致股东信、索罗斯演讲等公开语料，微调出具有他们语言风格和大致投资哲学倾向的对话模型。用户可将这些人格作为顾问，在辩论中使用。
- 用户也可以基于自己上传的交易记录和Skill，训练专属人格，并在社区中授权他人“雇佣”自己的AI顾问。

### 功能五：虚拟投资与Skill回测

- **虚拟投资实验室**：用户挑选任意社区Skill或自己训练的模型，设定初始资金和起止日期，系统在历史行情中逐日模拟交易（考虑滑点、手续费、涨跌停限制）。
- 输出完整绩效报告：净值曲线、年化收益率、阿尔法、贝塔、夏普比率、最大回撤、胜率、盈亏比。
- 支持**多Skill组合**：像基金组合一样分配资金，查看整体表现。
- “时光机”功能：选定历史上的真实事件（如2020年3月大跌），注入新闻情绪到Skill决策中，测试压力表现。
- 表现优异的虚拟投资策略可**一键克隆为实盘托管策略**，进入真实交易环境。

### 功能六：AI投资教育普及Agent

- 嵌入全平台的侧边栏助手“小智”，可随时唤醒。
- 具备**多模态解释**能力：用户截屏技术指标图，上传后Agent识别并解释（图像识别+K线知识库）。
- **板块地图导航**：交互式行业树，点击“半导体”直接展示产业链上游（设备、材料）、中游（设计、制造）、下游（封测、应用），并给出代表股。所有数据由行情Agent动态更新。
- **情景模拟问答**：“如果明天降准50BP，对银行和地产有什么影响？” 教育Agent会调用宏观分析模型，生成通俗解读和模拟推演。
- 与辩论系统联动：当某组辩论频繁提到“光伏”“钙钛矿”等概念，小智自动提示用户是否了解，并弹出知识点卡片。

## 五、数据流与存储设计

**核心数据流**：
1. 爬虫/行情网关 → Kafka原始数据Topic → 流处理服务（清洗、聚合、情感分析）→ 存入MySQL(元数据) + ClickHouse(时序行情) + Elasticsearch(新闻全文检索)。
2. 用户请求 → API网关 → 推荐服务 → 调用Agent编排器，从缓存/数据库拉取最新快照 → 启动辩论（调用模型池） → 结果写入Redis供前端轮询，并持久化至MongoDB。
3. 交易指令 → 风控微服务 → 同花顺Agent执行 → 返回成交写入交易记录表，同时推送至用户端。
4. 用户反馈 (👍/👎) → 反馈收集服务 → 存储至数据湖(MinIO)，夜间触发微调流水线，新LoRA权重上传模型注册中心，更新用户模型版本。
5. Skill回测请求 → 回测引擎 → 从ClickHouse抽取历史行情切片 → 模拟撮合，计算结果写入结果表。

**关键存储选型**：
- 关系型：PostgreSQL（用户、账户、订单、Skill元数据）
- 时序：ClickHouse（行情、回测净值）
- 文档：MongoDB（辩论记录、新闻详情、用户偏好卡）
- 向量库：Milvus（教育知识库、人格语料检索）
- 对象存储：MinIO / S3（微调权重、Skill代码包、截图）

## 六、技术栈建议

| 领域 | 技术选型 |

| 后端框架 | Python FastAPI + Go（高性能交易网关） |

| 消息队列 | Apache Kafka（行情、新闻流），RabbitMQ（交易指令） |
| 容器编排 | Kubernetes (K8s) + Helm |

| 安全 | 金融数据AES-256加密存储，交易指令双因素验证，敏感服务独立VPC |

## 七、商业模式

1. **基础免费版**：每日1次简易推荐，社区浏览，模拟回测限3年数据，教育助手基础问答。
2. **专业版订阅（¥99/月）**：解锁四组辩论全量报告、每日多次推荐、无限次回测、参与微调训练、绑定实盘托管（限额）。
3. **交易盈利分成**：对托管账户月度正收益部分收取5%~15%的业绩报酬（类似投顾模式，需持牌）。
4. **Skill市场交易**：用户可定价出售Skill使用权，平台抽成20%。热门人格订阅，创作者获分成。
5. **机构版API**：为基金/理财平台输出AI投研能力，按调用量计费。

## 八、合规与风险提示

- 明确声明**AI生成内容不构成投资建议**，托管交易须用户签署《自动化交易委托协议》及《风险揭示书》。
- 同花顺账户连接仅通过官方授权接口，密码及交易密码全程不可逆加密，平台不留存明文。
- 设置熔断机制：市场异常波动、账户单日亏损达限额立即暂停AI交易，转为人工决策。
- 所有模型训练数据需合法合规，不得使用内幕信息，社区Skill必须通过沙箱安全审核。

## 九、实施路线图

- **Phase 1 (MVP)**：完成新闻/行情爬虫、同花顺模拟交易连接、单模型推荐、基础教育Agent，验证核心链路。
- **Phase 2**：上线四组辩论系统、用户偏好微调向导、回测实验室、社区Skill上传与展示。
- **Phase 3**：实现实盘托管与精细风控、人格市场、多模型异构辩论优化，探索业绩报酬模式。
- **Phase 4**：开放机构API、全球化市场支持、引入强化学习自我博弈优化策略。
