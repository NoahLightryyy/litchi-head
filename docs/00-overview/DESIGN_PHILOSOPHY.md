# 🏛️ 设计哲学：虚拟小投行

> 这不是一个更好的交易机器人，而是一个组织的模拟。

---

## 一、为什么不是"超天才"

市面上的量化 Agent 大多追求一个目标：**训练一个超级交易员，能看盘、能分析、能决策、能赚钱。**

这是个人英雄主义模型。问题在于：一个人的盲点，就是整个系统的盲点。他看错了，全错。

人类大脑约 860 亿神经元，但有效决策带宽约 10 bit/s；大模型每秒处理数千 token。**单个神经元比 transformer 的一个参数复杂得多，但人在"并行维度"上完全不是对手。** 一个人再聪明，只能同时持有 3-7 个变量的心智模型；LLM 的上下文可以同时分析几十个维度还不忘。

### 1.b 还有一个更实际的问题：上下文的边界

> **2026-06-23 追加** — 来自实盘使用的观察。

LLM 能装几十个维度不忘，但金融市场的维度不是几十个，是成百上千个：

```
一份财报            → 利润表/资产负债表/现金流量表 × 3~5 年趋势 = ~15 个指标
上下游分析          → 供应商/客户/替代品/新进入者/竞争格局 = ~10 个维度
技术策略            → K 线形态 × N 种指标 × 多周期 = ~30 个维度
宏观环境            → 利率/汇率/通胀/政策/地缘 = ~10 个维度
资金流向            → 主力/散户/北向/融资融券 = ~8 个维度
行业对比            → 同行 N 家 × 核心指标 = ~20 个维度
情绪面              → 新闻/舆情/社交媒体 = ~5 个维度
```

**全部塞进一个模型 → 上下文被冲烂。** 不是模型不够聪明，是它一次能"认真看"的东西有限。把所有这些塞进一个 prompt，模型会在财报细节里忘了技术信号，在技术信号里丢了宏观判断。

**结论**：这不是"多模型比单模型好"的哲学问题，是一个**工程分治问题**——把金融市场的不同维度分给不同的 Agent，各看各的，最后汇总碰撞。就像投行里行业研究员不会同时做交易员的活。

**一个超天才干不过一个体系严密的系统。**

**结论**：用 LLM 模拟"一个人"是浪费它的带宽，用它模拟"一个组织"才是正途。

## 二、虚拟小投行：角色映射

投行是一个多角色分权制衡的系统。以下是从投行运作中提炼的、能套用到本项目的机制：

### 2.1 角色映射表

| 投行角色 | 本系统对应 | 能套用的机制 |
|:---------|:----------|:------------|
| 行业研究员 | 4 位分析师（基本面/技术面/情绪面/宏观面） | 每人独立出报告，结构化输出，不互相干扰 |
| 投资委员会 | 7 位大师辩论 + D3 评审投票 | 多视角碰撞，加权投票决策 |
| 投资组合经理 (PM) | PM 裁决层 | 综合全部上游产出，做最终决策 |
| 风控官 | 三层风控（激进/保守/中性）+ 交易纪律 | **独立否决权**，风控不参与分析只做判断 |
| 交易员 | T1 交易员 | 执行层面，不做决策判断 |
| 首席经济学家/策略师 | 宏观面分析师 | 提供宏观视野 |
| 研究合规官 (Supervisory Analyst) | 质量保障部 | 审计输出质量，检查夸大/虚假表述 |

### 2.2 不能套用的投行制度

| 投行制度 | 为什么不能套 |
|:---------|:-----------|
| 投行部 (IBD) | 本系统不做 IPO/并购/发债，不需要 |
| 销售 (Sales) | 没有"客户"要维护，用户就是自己 |
| 薪酬委员会/晋升制度 | AI Agent 不需要激励，不需要怕被裁 |
| 合规文书制度 | 没有监管红线需要应对 |
| 季度会议制度 | AI 之间不需要等"下季度"才能同步 |

### 2.3 能套用的投行机制（核心三原则）

#### 原则一：信息隔离 —— 只传结构化摘要，不传原始上下文

> **投行启示**：研究部和投行部之间有 Chinese Wall，目的是防利益冲突。
> **本系统借鉴**：虽然 AI Agent 不会为了利益撒谎，但全量传递上下文会导致 token 浪费和决策干扰。

每层 Agent 只传结构化的关键结论给下游，不传原始数据和思考过程：

```
分析师产出 → AnalystReport（结构化摘要，不含原始行情数据）
策略师产出 → VoteSummary（评级 + 置信度 + 关键依据）
辩论产出 → DebateSummary（共识点 + 分歧点）
```

**落地方式**：StateGraph 每层结束后剪裁 state，只保留该层的结构化 model，删除原始 prompt 和中间结果。

#### 原则二：偏斜公开 — 不追求绝对客观，而是公示偏斜方向

> **投行启示**：投行评级系统性偏乐观（Buy 远多于 Sell），有经验的买方知道"Hold ≈ 卖"。
> **本系统借鉴**：本系统的辩论引擎天然偏悲观（所有人互相找茬），

不试图消除偏斜（做不到），而是：
1. 每次辩论结束后统计偏斜方向（正面/负面观点比例）
2. 展示给用户："本次辩论偏斜度：偏悲观 23%"
3. 展示历史偏斜趋势："过去 20 次平均偏斜 -17%"
4. 用户基于偏斜信息自行校准判断

#### 原则三：复盘追责 —— 公式化、可量化、自动化

> **投行启示**：分析师连续错会被降级，连续对会拿更多奖金。
> **本系统借鉴**：AI Agent 不需要奖金也不会被开除，但系统应根据历史表现自适应调整。

**所有调整通过公式完成，不经过自然语言反思**：

| 可调旋钮 | 调整方式 | 当前状态 |
|:---------|:--------|:--------|
| 投票权重 | TrustTracker 准确率 → M4 权重系数 | ✅ 已做 |
| 发言顺序 | 准确率高的先发言，设定辩论基调 | 📋 待做 |
| 参与资格 | 准确率低于阈值的 Agent 本轮免战 | 📋 待做 |
| 置信度校准 | Agent 说"90%确信"但只对 55%，自动打 6 折 | 📋 待做 |
| 市场状态路由 | 不同市况下启用不同 Agent 组合 | 📋 待做 |

## 三、反馈机制：三权分立

### 3.1 公式负责调整

```
市场结果 → 量化指标（准确率/胜率/盈亏比）
        → 公式计算（不可调，写死的数学逻辑）
        → 调整旋钮（权重/顺序/参与资格）
        → 影响后续决策
```

**公式是硬编码的数学逻辑，不经过 LLM。** 不会出现"LLM 自己反思自己越改越偏"的问题。可审计、可回滚、可预测。

### 3.2 镜子负责展示

系统不替你做决定，而是把历史对比摊给你看：

```
辩论结束后，系统自动产出：
  📊 决策摘要（已有）
  📉 历史对比：上次类似情况，A 大师说了什么？后来结果如何？
  💡 系统提示：基于历史数据，"当 RSI < 30 时，B 大师的判断准确率只有 33%"

你看了之后，手动调权重或下次注意。
系统不会自作主张改任何东西。
```

### 3.3 人负责拍板

最终决策权在用户手里。系统做的是：**提供结构化的多维信息 + 历史经验参考 + 偏斜提示**，但买不买、买多少、什么时候买，不替用户决定。

## 四、需要新增的角色

### 4.1 灵感官（疯狂天才）

> 每个投行里都有一个人，想法天马行空，10 个里 9 个是错的，但那 1 个能赚 10 倍。

| 属性 | 值 |
|:-----|:----|
| 角色定位 | 反共识分析师，主动偏离主流逻辑 |
| 技术参数 | temperature = 0.9+，weight 天然较低 |
| Prompt 设定 | "说出一个主流观点认为疯狂、但你确实有理由相信的可能性" |
| 权重策略 | 天然低权重，但一旦命中大幅提升信任度 |
| 目的 | 拓宽搜索空间，不是追求正确 |

灵感官不参与辩论，不参与投票。他的产出独立展示在决策页面的"反共识观点"区，供用户参考。

| 维度 | TradingAgents | FundaPod | BestAITrader | **本项目** |

| 偏斜管理 | 无 | 无 | 无 | 🏆 **偏斜公示 + 用户自行校准** |
| 反馈机制 | 自然语言反思 | 无 | 后验复盘 | 🏆 **公式调整 + 镜子展示 + 人决策** |
| 测试覆盖 | 未公开 | 无 | 无 | 🏆 **946 测试 + CI 全绿** |
| 债务管理 | 无 | 无 | 无 | 🏆 **28 条技术债务跟踪** |
| 知识科普/反思 | 无 | 无 | 无 | 🏆 **镜子反思 + 历史决策学习** |

**核心差异不在技术架构，而在理念：这是一个有纪律的组织，不是一个聪明的交易员。**

### 不做的事

- ❌ 不做"Agent 晋升/裁撤"等拟人化制度
