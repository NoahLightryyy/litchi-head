## 填写示例

以下是一个填写完整的示例供参考：

```markdown
###### TD-009 示例：API 密钥未加密存储

| 属性 | 值 |
|------|-----|
| **分类** | `🔒 security` `severity:major` `module:utils` `impact:安全性` |
| **发现日期** | 2026-06-05 |
| **发现人** | AI 安全审计 |
| **状态** | `🆕 待评估` |
| **本金估算** | ∼3h |
| **日利息** | 每延迟一天，API Key 泄露风险就多一天 |

**描述**：
`.env` 文件中的 API Key 以明文存储，虽然被 `.gitignore` 排除，
但本地开发时仍可能被恶意软件或误操作泄露。

**具体问题**：
1. ❌ `.env` 文件中的 `DEEPSEEK_API_KEY` 是明文
2. ❌ 日志中可能意外打印 API Key
3. ❌ 没有密钥轮换机制

**利息分析**：
- 当前只在本地开发，风险可控
- 未来多人协作或 CI 中使用真实密钥时风险骤增

**修复方向**：
- 短期：在日志过滤器中屏蔽密钥输出
- 长期：集成密钥管理服务（Vault / AWS Secrets Manager）
```
