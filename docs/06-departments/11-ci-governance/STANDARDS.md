# 📐 CI 治理部技术规范

> 本文件定义 CI 治理部自身的标准。全局 CI 门禁标准见 `docs/01-guides/ci/STANDARDS.md`。

---

## 1. CI 问题管理标准

### 问题生命周期

```
发现 → 登记 (ISSUES.md / DEBT.md) → 定位 → 修复 → 验证 → 复盘 → 关闭
```

### ID 命名

- CI 系统问题：`CI-NNN`（如 CI-001）
- 与其他部门债务关联：在关联部门 DEBT.md 中加引用

### 严重度分级

| 级别 | 定义 | 响应时间 |
|:----:|:-----|:--------:|
| P0 🔴 | 所有版本失败（3.12 + 3.13 全红） | **立即**，当前批次内 |
| P1 🟡 | 单版本失败 | **当前批次内** |
| P2 🟢 | 非阻塞项（pip-audit 警告、3.13 超时） | **本次迭代内** |

## 3. 跨部门协作标准

### CI 红了，谁修？

```
CI 红了
├── 根因在 CI 系统本身（runner 挂了、超时太短、配置错误）
│   └── → CI 治理部修
├── 根因在基础设施（依赖装不上、Python 版本不对）
│   └── → 通知基础设施部
└── 根因在业务代码（测试失败、类型错误、lint 报错）
    └── → 通知对应部门修
          └── CI 治理部负责"追"——红了没修 → CI 治理部的债
```

### 数据契约变更通知

CI 治理部不生产数据契约，但 CI 配置变更（检查项增减、超时调整）影响所有部门。
