**实现内容**：
在 `review_round → aggregate` 之间插入 `review_report` 节点。每位大师完成第一轮分析和第二轮交叉反驳后，由独立的 LLM 评审人格对辩论整体质量做出结构化评估。输出 `IndependentReview`，包含：
- 裁判独立评级/评分（基于全部信息）
- `weight_suggestions` — 对每位大师的权重调整建议（乘数）
- 集体偏见识别和盲区提示
- 关键风险综合和分析一致性观察

**聚合升级**：`aggregate_node` 自动检测 `review_report` 的 `weight_suggestions`，作为置信度乘数影响 `weighted_score`。有 rebuttal + weight_suggestions 时优先应用 rebuttal 调整，再应用权重乘数。

### 架构变更图

```
collect_data → master_round → review_round (D1) → review_report (D3 新增) → aggregate (D3 升级) → END
```

### 向后兼容
- `review_report=None` 时 aggregate 行为不变
- `weight_suggestions={}` 时加权计算与原行为完全一致

### 代码审查追踪
- 独立评审失败时返回 `IndependentReview()` 默认值，不阻塞流程
- 所有大师失败时返回空评审（quality=0, score=0）
- `rerun/review_round` 可选，无后者时评审只看大师分析
