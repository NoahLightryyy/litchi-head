### 1. 多设备 Python 环境对齐

**背景**：litchi-head 项目在不同设备间迁移时发现 Python 3.11（`python` 命令）和 Python 3.13（`pip3` 命令）指向不同解释器，导致依赖包安装到 3.13 但运行时使用 3.11。

**修复内容**：
- 向 Python 3.11 安装缺失依赖：`langgraph`、`akshare`、`langchain-deepseek`、`langchain-openai`、`langchain-anthropic`、`pytest-asyncio`

**涉及文件**：无代码修改，仅环境操作

### 2. 修复 `langchain_anthropic` 模块级硬导入（TD-001 优化）

**涉及文件**：
- 🔧 `src/utils/llm.py` — `ChatAnthropic` 从模块级 import 改为惰性导入

**内容**：
`src/utils/llm.py` 第 28 行 `from langchain_anthropic import ChatAnthropic` 是模块级硬导入。
当 `langchain-anthropic` 未安装时，所有引用 `llm.py` 的模块（`MasterAgent`、`DebateOrchestrator`、`XiaoZhiAgent` 等）都在 import 阶段 cascade 崩溃。

修复方案：将 `ChatAnthropic` 移到 `_build_llm()` 的 `case "anthropic"` 分支内惰性导入，
并在缺失时抛出清晰的安装指引错误。不影响已有逻辑（`DeepSeek` 和 `OpenAI` 路径保持不变）。

**影响**：修复前 6 个测试模块 collection 失败 → 修复后无 collection 错误

### 3. 修复测试断言鲁棒性（Windows 精度）

**涉及文件**：
- 🔧 `tests/test_agents_base.py` — `latency_ms > 0` → `>= 0`
- 🔧 `tests/test_debate_orchestrator.py` — `total_latency_ms > 0` → `>= 0`

**内容**：
Windows 上 `time.monotonic()` 分辨率有限，mock 调用太快时两次调用间差为 `0.0`。
