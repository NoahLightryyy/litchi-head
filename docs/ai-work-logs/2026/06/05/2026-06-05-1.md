### 1. 实现 `src/utils/llm.py` — LLM 调用封装层

**涉及文件**：
- ⭐ `src/utils/llm.py` — 新增 LLM 调用封装（核心文件）
- 🔧 `pyproject.toml` — Pyright 配置指向 Python 3.13
- 🔧 `docs/技术债务与架构决策/技术债务日志.md` — TD-001 状态更新

**内容**：
实现了 TD-001 要求的 LLM 调用封装层，包含三个核心接口：

| 接口 | 签名 | 功能 |
|------|------|------|
| `llm_service.get_llm(provider)` | `(str\|None) -> BaseChatModel` | 按 provider 获取/缓存 LLM 实例 |
| `llm_service.ainvoke(prompt, ...)` | `(str, ...) -> str` | 纯文本调用，自动重试+费用记录 |
| `llm_service.invoke_structured(prompt, output_model, ...)` | `(str, type[BaseModel], ...) -> BaseModel` | 结构化输出，自动重试+费用记录 |

**关键实现细节**：
- `ChatDeepSeek`/`ChatOpenAI` 工厂函数，通过 `settings.llm_provider` 切换
- `with_structured_output()` 自动解析 LLM 输出为 Pydantic 模型
- `tenacity` 指数退避重试（retry 3 次，2-30s 等待）
- `_record_usage()` 自动集成 `cost_tracker` 记录 token 费用
- 实例缓存：相同 provider 复用同一 LLM 实例

**质量检查**：
- Ruff: ✅ 0 errors
- Pyright: ✅ 0 errors
