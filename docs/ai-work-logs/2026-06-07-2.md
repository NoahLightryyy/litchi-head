### 2. KnowledgeBase 核心实现

**涉及文件**：
- ⭐ `src/memory/knowledge_base.py` — 全新模块
- ⭐ `tests/test_memory_knowledge_base.py` — 15 个业务测试

**设计决策**：

为 MVP 快速落地，使用 **n-gram 字符特征 + TF 向量** 做语义搜索，零外部依赖：

| 维度 | 当前实现 | 未来升级（Phase 1） |
|------|---------|-------------------|
| 向量化 | n-gram TF（n=2~4，字符级） | FAISS + BGE-small-zh |
| 相似度 | 余弦相似度 | 同上 |
| 外部依赖 | numpy（已有） | faiss-cpu + sentence-transformers |

**核心能力**：
- `ingest(file_path, type)` — 按 `##` Markdown 标题自动分块，5000 词表自动构建
- `search(query, k, type_filter, min_score)` — 余弦相似度 top-k，支持类型过滤
- `save()` / `load()` — 持久化到 `chunks.json` + `embeddings.npy` + `vocab.json`
- `ingest_directory()` — 批量导入目录

**测试覆盖**（15 tests）：
- 初始化（2）、导入（5）、搜索（4）、持久化（3）、边界（1）

### 5. 教育小智 Agent（XiaoZhiAgent）

**涉及文件**：
- ⭐ `src/agents/xiao_zhi.py` — 全新 Agent
- ⭐ `tests/test_agents_xiao_zhi.py` — 15 个测试

**工作流程**：
1. 接收用户问题（`input_data["question"]`）
2. KnowledgeBase 语义检索 top-3 相关知识
3. 知识上下文 + 系统 prompt → LLM（经过 `llm_service` 层）
4. 返回结构化结果（`answer` + `knowledge_sources`）

**关键设计**：
- **降级策略**：知识库未命中时，降级为纯 LLM 回答（标注"基于我的理解"）
- **confidence**：知识命中 0.9，未命中 0.6
- **系统 prompt**：定义教育小智角色（产品名"小智"），强调五条核心原则
- 所有 LLM 调用必经 `src/utils/llm.py` ✅ 符合架构红线
