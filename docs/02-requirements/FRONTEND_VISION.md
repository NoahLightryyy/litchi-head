# 前端视觉方向 —— 调研报告与设计方案

> 基于 TradingView / Bloomberg / Robinhood / 东方财富 / 同花顺 / Webull 等主流交易平台调研，
> 结合 2025-2026 AI 交易平台趋势，为 litchi-head 前端 MVP+ 确定视觉方向。

---

## 1. 调研结论

### 1.1 标杆平台核心特征

| 平台 | 视觉风格 | 最值得学的地方 |
|:-----|:---------|:-------------|
| **TradingView** | 暗色，蓝绿配色，云同步 | K 线图表质量、指标系统、Pine Script |
| **Bloomberg 终端** | 极暗背景，数据密集，等宽字体 | "信息密度 = 价值"的心理暗示 |
| **Robinhood** | 极简白底，大字体，圆角 | 新手入门流畅度，操作 < 3 步 |
| **东方财富** | 信息量大，多面板 | 一站式覆盖，主力资金可视化 |
| **雪球** | 社区风格，卡片式 | 内容化投资，UGC 和决策结合 |

### 1.2 2025-2026 关键趋势

1. **AI 对话式交互** — eToro Tori / Quant / 同花顺问财，从"点菜单"转向"问一句话"
2. **AI Agent 自主决策** — eToro Agent Portfolio / Bitget Agent Hub，用户定规则 AI 执行
3. **量化工具平民化** — 机构级分析工具以 $10-20/月 提供给散户
4. **暗色模式已是标配** — 没有暗色的交易平台会被认为"不专业"
5. **数据密度回归** — Bloomberg 风格"每个像素都有意义"正在回归

### 1.3 litchi-head 的定位

**核心差异化**：
- 不是"另一个看盘软件"——而是 **AI 驱动的自上而下决策漏斗**
- 不是"AI 助手"（帮你看数据）——而是 **AI 多 Agent 辩论决策**（帮你做决策）
- 这正踩在全球最前沿的 **Agentic AI + 金融决策** 方向

**产品使命**：
> 散户打开 litchi-head，10 秒内感受到"这是个专业工具"，
> 30 秒内理解自上而下的决策流，60 秒内拿到一个 AI 辩论结论。

## 2. 视觉系统设计

### 2.1 情绪板

```
专业 · 冷静 · 数据驱动 · 可信
───────────────────────────────
👔 像 Bloomberg 终端一样专业
🌙 像 TradingView 一样舒适暗色
🧠 像真正的 AI 一样理性不花哨
```

### 2.2 色彩体系

```css
/* ── 基础色板 ── */
--bg-primary:    #0D1117;    /* 主背景（极深黑） */
--bg-secondary:  #161B22;    /* 卡片/面板背景 */
--bg-tertiary:   #21262D;    /* 悬浮/选中态 */

--text-primary:   #E6EDF3;   /* 主文本 */
--text-secondary: #8B949E;   /* 次要文本 */
--text-muted:     #484F58;   /* 不可用/占位 */

/* ── 功能色（TradingView 风格） ── */
--accent-blue:   #2962FF;    /* 主交互色 */
--accent-green:  #00C853;    /* 涨/买入/看多 */

--accent-gold:   #FFB300;    /* 警告/关注 */

/* ── 图表色 ── */
--chart-bull:    #26A69A;    /* 阳线 */

```

### 2.3 排版系统

```css

/* 层级 */

--text-2xl: 2rem;      /* 超大数字 */
--text-3xl: 2.5rem;    /* 主价格 */
```

### 2.4 间距 & 圆角

## 4. 技术选型推荐

### 4.1 分阶段策略

```
Phase 1 (MVP — 现在)        →  Phase 2 (专业版 — 下一阶段)     →  Phase 3 (产品化)
─────────────────────────────────────────────────────────────────────────────────
Streamlit + Plotly           React + Next.js 16 + Tailwind v4  全面 TradingView 生态
.stremlit/config.toml 主题   shadcn/fintech 模板                自定义设计系统
Plotly K 线图                Lightweight Charts                  TradingView Charting Lib
akehare 实时缓存              FastAPI 中间层 + WebSocket           WebSocket 流
```


### 4.2 为什么 Phase 1 仍建议 Streamlit

原因很简单——**核心逻辑验证优先于视觉**。先用 Streamlit 把自上而下的漏斗逻辑跑通：

1. 宏观总览页 → 板块排行 + AI 简报
2. 板块/产业链页 → 产业链地图 + 个股列表
3. 个股决策页 → K 线 + AI 辩论

这些逻辑验证了，再花精力升级到 React 做专业 UI，一举两得。

## 6. 关键设计决策

| 决策 | 选择 | 理由 |
|:-----|:-----|:------|
| **暗色还是亮色** | 暗色默认，亮色可选 | 交易平台行业标准，专业感强 |
| **信息密度** | Bloomberg 风格（中高密度） | 散户需要信息量来感受"专业有价值" |
| **对话式 AI** | 暂不引入，先做面板式 | AI 辩论作为可触发的面板，比对话流更适合股票分析场景 |
| **页面跳转 vs 单页滚动** | 三页跳转（MVP） | 每页聚焦一个决策层级，逻辑清晰 |
| **移动端** | MVP 专注桌面 | 桌面体验做好后再适配移动端 |
